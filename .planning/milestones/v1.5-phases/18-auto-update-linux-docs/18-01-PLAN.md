<objective>
Wire the Tauri updater plugin infrastructure: Cargo dependencies, plugin registration in lib.rs, tauri.conf.json config (createUpdaterArtifacts + plugins.updater placeholder), and capability permissions.

Purpose: This is the foundational Rust/Tauri config layer that makes auto-update possible. Without these four changes, tauri-action silently skips .sig upload, the update check fails at runtime, and the frontend cannot call downloadAndInstall() or relaunch().

Output: Compilable Rust codebase with updater and process plugins wired. tauri.conf.json ready for pubkey injection in Plan 04. capabilities/default.json grants the required permissions.
</objective>

  "active": true,
  "targets": "all",
  "createUpdaterArtifacts": true,
  "icon": [

CRITICAL: `"createUpdaterArtifacts"` MUST be inside `"bundle"` — not at the top level. Without this, tauri-action silently skips `.sig` upload and `latest.json` never gets signature entries.

2. Add a new top-level `"plugins"` key at the same level as `"bundle"` and `"app"`:

}

The pubkey value `"UPDATER_PUBKEY_PLACEHOLDER"` is intentionally a placeholder string. Plan 04 will replace it with the real Ed25519 public key content after key generation. The placeholder allows `cargo check` to pass without a real key.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tauri-action → release | CI-signed artifacts uploaded via GITHUB_TOKEN; signature covers binary, not JSON manifest |
| tauri.conf.json → plugin-updater | Public key embedded in committed config; must be public key only, never private |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | tauri-plugin-updater download | mitigate | Ed25519 signature verification built into tauri-plugin-updater; app refuses to install unsigned artifacts automatically |
| T-18-02 | Information Disclosure | tauri.conf.json pubkey field | accept | Public key is intentionally public; private key lives only in GitHub Actions secret TAURI_SIGNING_PRIVATE_KEY — never committed |
| T-18-03 | Tampering | Update endpoint (majesnix/tap/releases/latest/download/latest.json) | mitigate | Signature verification covers the downloaded binary, not latest.json — a spoofed latest.json cannot deliver unsigned code |
| T-18-04 | Spoofing | latest.json manifest | accept | Manifest is not trusted for code execution; tauri-plugin-updater checks the .sig file against the embedded pubkey before executing any update |
| T-18-05 | Denial of Service | check() network failure on startup | mitigate | All check() calls wrapped in .catch() — update failure must not affect app functionality (Pattern 5, Pitfall 4 in RESEARCH.md) |
</threat_model>
