## v1.1 Requirements (All Delivered)

### Theme

- [x] **DRK-01**: App detects the OS dark/light preference (`prefers-color-scheme`) on startup and applies the corresponding theme automatically when the user's mode is set to "system"
  - **Outcome:** Delivered by Plan 05-01 — next-themes ThemeProvider with `enableSystem` applies OS preference via `matchMedia`. Verified by human UAT.

- [x] **DRK-02**: User can switch between three theme modes — system, light, and dark — via a toggle control in the app UI
  - **Outcome:** Delivered by Plan 05-02 — ThemeToggle icon cycle button (Monitor/Sun/Moon) in sidebar footer, CYCLE_ORDER drives stateless progression.

- [x] **DRK-03**: The selected theme mode persists across app restarts (stored via tauri-plugin-store, same mechanism as connection profiles)
  - **Outcome:** Delivered by Plan 05-01 — ThemeBootstrap reads tauri-plugin-store on mount (cross-restart authority), race guard via bootstrapped flag prevents stale localStorage clobber.

- [x] **DRK-04**: All existing UI surfaces render correctly in dark mode — form panel, connection sidebar, publish bar, AMQP properties sheet, message history panel, response tab, modals, and all shadcn/ui components
  - **Outcome:** Delivered by Plan 05-03 — human visual UAT approved 2026-05-18. Dark CSS variables in index.css were already complete; UAT confirmed integration with next-themes produces correct visuals across all 30+ surface items.
