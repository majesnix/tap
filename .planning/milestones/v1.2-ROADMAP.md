# Milestone v1.2: Form Improvements

**Status:** ✅ SHIPPED 2026-05-19
**Phases:** 6–8
**Total Plans:** 7

## Overview

Extended the Tap dynamic form renderer to cover the remaining major proto field types and added a JSON override mode for power users who need fine-grained control over outgoing message values.

Three field types were implemented across three phases: a bytes field with RFC 4648 base64 validation, a map field with typed key-value rows and duplicate detection, and a JSON override toggle with two-way form ↔ editor sync. The milestone also introduced a quick-task fix (260519-q01) to restore the send-button blocking gate that had been broken by an incorrect code review finding in Phase 7.

## Phases

### Phase 6: BytesField

**Goal:** Users can fill in bytes fields using standard base64 with immediate feedback on invalid input
**Depends on:** Nothing (self-contained frontend change, zero Rust changes)
**Requirements:** BFLD-01, BFLD-02, BFLD-03, BFLD-04
**Plans:** 1 plan

Plans:
- [x] 06-01-PLAN.md — Create BytesField component (TDD), wire into ProtoFormRenderer, remove bytes branch from ScalarField

**Details:**

BytesField is a new React component implementing RFC 4648 base64 input for proto `bytes` fields. Key elements:
- **Two-layer zod validation:** char-set regex rejects URL-safe (`-`/`_`) characters with an inline error; `.refine()` checks structural validity (padding, length divisibility by 4)
- **UTF-8 text helper:** Controlled-open Popover with a textarea — user types UTF-8 text, `TextEncoder`-based `utf8ToBase64` helper encodes it (safe for non-ASCII; no `btoa()` on user-supplied text)
- **Byte count label:** displayed only after `safeParse().success` — avoids `InvalidCharacterError` from `atob()` on partially-typed values
- **ProtoFormRenderer pre-dispatch branch:** bytes check added before the frozen switch body; ScalarField bytes handling removed (clean separation)

### Phase 7: MapField

**Goal:** Users can add, edit, and remove entries for `map<K, V>` proto fields, which render as typed key-value rows and encode as binary protobuf wire format
**Depends on:** Phase 6 (establishes base64 alphabet convention for bytes-typed map values)
**Requirements:** MFLD-01, MFLD-02, MFLD-03, MFLD-04, MFLD-05
**Plans:** 4 plans

Plans:
- [x] 07-01-PLAN.md — Rust layer: FieldKind::Map variant, is_map() extractor guard, Value::Map encoder + 4 unit tests (Wave 1)
- [x] 07-02-PLAN.md — TypeScript wiring: FieldKind map union member, ProtoFormRenderer pre-dispatch branch, buildDefaultValues case (Wave 2)
- [x] 07-03-PLAN.md — MapField component: full TDD implementation — useFieldArray rows, key dispatch, duplicate detection, guard field, renderValue prop (Wave 3)
- [x] 07-04-PLAN.md — Human verify: live app verification of all MFLD-01 through MFLD-05 requirements (Wave 4)

**Details:**

MapField is a full-stack feature requiring coordinated Rust + TypeScript changes across 4 waves:

**Rust (07-01):**
- `FieldKind::Map { key_type: ScalarKind, value_kind: Box<FieldKind> }` variant in `types.rs`
- `is_map()` guard in `extract_field_schema` (before `is_list()` check) — prevents map fields from falling through to Kind::Message
- `is_map_entry()` filter in `extract_schema` removes synthetic `XxxEntry` types from the message list
- `Value::Map(HashMap<MapKey, Value>)` encoding path with `json_to_map_key` helper covering all proto3 key types
- `scalar_or_message_value_for_map_entry` handles enum and message value types through the map entry descriptor
- 4 Rust unit tests: string key, int32 key, bool string key, empty array

**TypeScript wiring (07-02):**
- `{ type: "map"; key_type: ScalarKind; value_kind: FieldKind }` union member in `types.ts`
- `if (field.kind.type === "map")` pre-dispatch branch in `ProtoFormRenderer.tsx`
- `case "map": defaults[field.name] = []` in `buildDefaultValues`

**MapField component (07-03):**
- `useFieldArray` for add/remove row operations; rows stored as `Array<{key, value}>` (never `Record<K,V>` — silent JS key deduplication would lose data)
- Key input dispatched by `key_type`: `is32BitInt()` → `type="number"`, `is64BitInt()` → `type="text"` + regex pattern, `"bool"` → shadcn `Select` with true/false options, `"string"` → plain text input
- Duplicate detection via `useWatch + useMemo`; inline error displayed when `duplicateKeys.size > 0`
- Guard field mechanism: `register(guardName, { validate: () => hasDuplicatesRef.current ? "Duplicate key" : true })` + `trigger(guardName)` keeps `formState.isValid` false while duplicates exist (fixed by quick task 260519-q01 after Phase 7 code review incorrectly replaced this with `setError`)
- `renderValue` prop delegates to `ProtoFormRenderer.renderField` for full value-type support

**Human UAT (07-04):** All 7 steps approved in live Tauri app against real RabbitMQ. Additional human UAT (07-HUMAN-UAT.md) confirmed enum-valued and message-valued map entries encode correctly.

**MFLD-03 fix (quick task 260519-q01):** Phase 7 REVIEW-FIX IN-04 incorrectly removed the virtual guard field, replacing it with `setError`/`clearErrors`. This does not reliably keep `formState.isValid = false` in RHF `mode: onChange`. Fixed by restoring the `register+trigger` pattern and adding `!encodeError` to `PublishBar.canSend`.

### Phase 8: JSON Override Toggle

**Goal:** Users can switch between form view and a raw JSON editor, with two-way sync that preserves repeated and map field row counts
**Depends on:** Phase 6 (base64 string shape in latestValues), Phase 7 (map as Array<{key,value}> in latestValues)
**Requirements:** JSON-01, JSON-02, JSON-03, JSON-04, JSON-05, JSON-06
**Plans:** 2 plans

Plans:

**Wave 1:**
- [x] 08-01-PLAN.md — Install CodeMirror packages, export buildDefaultValues, implement JsonEditor component (TDD)

**Wave 2** *(blocked on Wave 1)*:
- [x] 08-02-PLAN.md — FormPanel JSON toggle integration (TDD) + human verify

**Details:**

**JsonEditor (08-01):**
- New React component wrapping `@uiw/react-codemirror` with `@codemirror/lang-json` syntax highlighting
- `theme` prop accepts `resolvedTheme` from `next-themes` — dark/light mode already in use across the app
- Controlled component (`value`/`onChange` props); tested with a textarea stub in jsdom
- `buildDefaultValues` exported from `ProtoFormRenderer.tsx` for use by FormPanel

**FormPanel JSON toggle (08-02):**
- Braces `Button` (ghost/icon-sm) added to FormPanel header with `aria-pressed` and `aria-label` for accessibility
- Local React state: `isJsonMode`, `entrySnapshot`, `jsonDraft`, `parseError` (not Zustand — JSON mode is session-only)
- `handleToggle` FORM→JSON: captures `latestValues ?? {}` as `entrySnapshot` (point-in-time snapshot), sets `jsonDraft`
- `handleToggle` JSON→FORM: `JSON.parse(jsonDraft)` with try/catch; detects unknown top-level keys via `message.fields` set; strips unknowns; merges with `buildDefaultValues`; signals via `setPendingReplayValues(mergedValues)` — reuses existing HIST-02 replay signal to trigger form reset
- Invalid JSON: inline error banner with Fix JSON (clears error, stays in JSON mode) / Discard (restores entrySnapshot, exits JSON mode)
- Unknown field warning: `toast.warning` listing field names not in the proto schema (non-blocking)
- `useTheme` called at top of component (before early-return guards — React hooks rule)

## Milestone Summary

**Quick Tasks:**
- 260519-q01 (2026-05-19): Fix MFLD-03 — restore send-button blocking on duplicate map keys (register+trigger + PublishBar encodeError gate)

**Key Decisions:**
- Map rows stored as `Array<{key, value}>` via `useFieldArray` — never as `Record<K,V>` (JS object keys deduplicate silently)
- ProtoFormRenderer dispatch switch is FROZEN — new field types added as pre-dispatch `if` branches before the switch
- RHF `register(name, { validate }) + trigger(name)` is the correct pattern for virtual guard fields; `setError` on unregistered fields does not reliably affect `formState.isValid` in `mode: onChange`
- JSON-to-form sync reuses `setPendingReplayValues` signal (HIST-02 path) — avoids calling `resetRef.current()` directly (ref is null until ProtoFormRenderer remounts)
- `entrySnapshot` captured at toggle entry from `latestValues ?? {}` — Discard always restores this snapshot, never re-reads current `latestValues`
- Two-layer duplicate detection: RHF `formState.isValid` gate (register+trigger) + `encodeError` in `PublishBar.canSend` (Rust WR-02 fallback)

**Issues Resolved:**
- MFLD-03 send-block broken by Phase 7 REVIEW-FIX IN-04 — restored via quick task 260519-q01
