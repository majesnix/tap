## Color

All tokens are CSS custom properties from `src/index.css`. Both light and dark modes are defined; the color system adapts automatically. Values below are light mode oklch references.

| Role | CSS Token | Light Value | Dark Value | Usage |
|------|-----------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` | `oklch(0.145 0 0)` | App background, toolbar surface |
| Secondary (30%) | `--muted` | `oklch(0.97 0 0)` | `oklch(0.269 0 0)` | Feed header (`bg-muted`), badge fills |
| Accent (10%) | `--primary` | `oklch(0.205 0 0)` | `oklch(0.922 0 0)` | Primary CTA button (Start Subscribe only) |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` | `oklch(0.704 0.191 22.216)` | Error status badge dot |

**Accent reserved for:** The Start Subscribe button (`variant="default"`) only. The Stop button uses `variant="outline"`. The mode toggle active state uses the existing radix ToggleGroup pressed style (maps to `bg-primary`). No other elements use the accent/primary color.

**Status badge color semantics** (dot indicator, reusing established `ResponseQueuePicker` pattern of `Badge variant="outline"` + colored dot + text label):

| Stopping | `bg-amber-500` | `outline` | Stopping |
| Error | `bg-destructive` | `destructive` | Error |

Pattern: `<Badge variant="outline"><span className="w-2 h-2 rounded-full {dot-class}" />{label}</Badge>`

## Copywriting Contract

| Element | Copy | Mode/State |
|---------|------|------------|
| Mode toggle — Drain segment | `Drain` | Always |
| Mode toggle — Subscribe segment | `Subscribe` | Always |
| Primary CTA — start | `Start` | Subscribe mode, Idle state |
| Primary CTA — stop | `Stop` | Subscribe mode, Running state |
| Primary CTA — stopping (disabled) | `Stop` (disabled) | Subscribe mode, Stopping state |
| Status badge — Idle | `Idle` | Subscribe toolbar, Idle state |
| Status badge — Running | `Running` | Subscribe toolbar, Running state |
| Status badge — Stopping | `Stopping` | Subscribe toolbar, Stopping state |
| Status badge — Error | `Error` | Subscribe toolbar, Error state |
| Empty state — Drain mode | `Select a queue and click Drain` | Drain mode, 0 messages |
| Empty state — Subscribe Idle, 0 messages | `Click Start to begin subscribing` | Subscribe mode, Idle, 0 messages |
| Empty state — Subscribe Running, 0 messages | `Waiting for messages…` | Subscribe mode, Running, 0 messages |
| Empty state — Subscribe Error | `{subscribeError} — Stop and try again` | Subscribe mode, Error state |
| Toast — consumer error (from channel) | `Subscribe error: {message}` | On DrainResult.error received via channel |
| Toast — queue empty (Drain) | `Queue is empty` (existing Phase 13 copy, unchanged) | Drain mode only |
| Toast — feed cap (Drain) | `Feed capped at 500 — {N} older message(s) removed` (existing, unchanged) | Both modes |
| Tooltip — Start disabled (no queue) | `Select a queue to start subscribing` | Subscribe mode, no queue selected |
| Tooltip — Start disabled (not connected) | `Connect to a RabbitMQ profile to subscribe` | Subscribe mode, disconnected |
| Tooltip — mode toggle disabled | `Stop the active session before switching modes` | Toggle, Running or Stopping state |

**Destructive actions:** None in this phase. Stop is reversible (user can restart). No confirmation dialog required.

**"Stopped — X messages received" toast:** Omitted. Auto-stop on profile switch is silent (D-12); adding a toast only for manual stop creates UX inconsistency. Silence is consistent.

Source: CONTEXT.md D-04, D-05, D-06; RESEARCH.md §Claude's Discretion; existing Phase 13 copy in `MessageFeedTab.tsx` line 106.

## Interaction States

### Mode Toggle

| State | Behavior |
|-------|----------|
| Default (Drain active) | Drain segment visually pressed; Subscribe segment inactive |
| Subscribe active | Subscribe segment visually pressed; drain count input + Drain button hidden; SubscribePanel shown |
| Disabled (Running/Stopping) | Both segments non-interactive; cursor `not-allowed`; tooltip on hover |

### Start Button

| State | Visual | Behavior |
|-------|--------|----------|
| Enabled (Idle, queue + decode types selected, connected) | `variant="default"`, Play icon | Calls `startSubscribe`; sets status to Running |
| Disabled — no queue | `variant="default"` disabled | Tooltip: "Select a queue to start subscribing" |
| Disabled — not connected | `variant="default"` disabled | Tooltip: "Connect to a RabbitMQ profile to subscribe" |
| Disabled — no decode types | `variant="default"` disabled | Tooltip: "Select at least one decode type" |
| Hidden | — | Hidden when `subscribeStatus === "Running"` or `"Stopping"` |

### Stop Button

| State | Visual | Behavior |
|-------|--------|----------|
| Enabled (Running) | `variant="outline"`, Square icon | Calls `stopSubscribe`; sets status to Stopping |
| Disabled (Stopping) | `variant="outline"` disabled, Loader2 animate-spin | No action; user waits for invoke to resolve |
| Hidden | — | Hidden when `subscribeStatus === "Idle"` or `"Error"` |

### Auto-Stop

No visual change beyond status badge transitioning Stopping → Idle. Feed messages are preserved (D-12). No toast on auto-stop.
