# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-18
**Phases:** 1–4
**Total Plans:** 18

## Overview

Tap v1.0 delivers the complete MVP: a Tauri desktop application (Rust backend + React frontend) that lets developers load `.proto` files, generate a dynamic form from the schema, connect to RabbitMQ, and send binary-encoded protobuf messages — plus read and decode incoming response messages — all without writing any code. Built as a team dev-tool: each developer installs it locally and uses their own saved connection profiles.

**Core Value:** Send a real protobuf message to RabbitMQ in under 30 seconds from a raw `.proto` file — no code, no curl, no manual encoding.

## Phases

### Phase 1: Proto Parsing + Form

**Goal:** User can load a `.proto` file, see a fully rendered type-aware form, fill in values, and inspect the resulting binary-encoded protobuf payload — entirely offline, no RabbitMQ required.
**Depends on:** Nothing
**Plans:** 6 plans

Plans:
- [x] 01-01: Walking Skeleton — full Tauri scaffold, Rust proto backend (parse_proto + encode_message), React form renderer, hex preview panel
- [x] 01-02: Scalar Field Full Implementation — all 16 scalar kinds, zod validation, zero-value defaults
- [x] 01-03: Nested + Repeated + Depth cap — NestedMessageField (collapsible, depth), RepeatedField (useFieldArray), DepthCapPlaceholder at 5 levels
- [x] 01-04: Enum + Oneof — EnumField (Select, name/number split), OneofField (RadioGroup, conditional mount, unregister)
- [x] 01-05: WellKnownType controls + include path persistence — WellKnownTypeField (Timestamp/Duration/fallback), IncludePathDialog with tauri-plugin-store
- [x] 01-06: Gap closure — fix broken debounce gate in FormPanel (useState reactive source + useEffect encoding pipeline)

**Requirements delivered:** PROT-01, PROT-02, FORM-01 through FORM-09

**Success Criteria verified:**
1. User opens a `.proto` file with nested imports and sees a fully rendered form within 2 seconds — no terminal, no compilation step.
2. Every field type (scalar, nested message, repeated, enum, oneof) renders correctly and accepts or rejects input according to proto type constraints.
3. Filling out the form produces a hex preview of the protobuf wire bytes that changes when field values change.
4. Recursive message types do not crash — renderer stops at 5 levels deep.

**Requirements delivered:** RESP-01 through RESP-05

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Tauri 2.x over Electron | Lighter bundle, Rust backend handles proto encoding and AMQP natively | ✓ Good — native feel, no Node.js overhead in production binary |
| Binary proto wire format | Matches production consumer expectations | ✓ Good — prost-reflect encoding validated against real RabbitMQ consumers |
| Runtime .proto parsing (protox + prost-reflect) | No pre-compilation step — developer just drops in the file | ✓ Good — protox Compiler API handled include path resolution cleanly |
| Save named connection profiles with keychain | Team members connect to different RabbitMQ instances, passwords need security | ✓ Good — OS keychain never exposed password to frontend or logs |
| reqwest + Management API for queue listing | AMQP 0-9-1 has no enumeration operation | ✓ Good — Management API approach worked; 401 surfaced as auth error not silent fallback |
| react-hook-form + zod (not RJSF) | Proto oneof/repeated don't map cleanly to JSON Schema | ✓ Good — useFieldArray handled repeated fields well; zod pinned to ^3.24.2 (v4 incompatible) |
| zustand 5.x for global state | Simpler than Redux for this scope | ✓ Good — 4 stores (proto, connection, amqp, history, response) stayed manageable |
| shadcn/ui nova preset | Zero-dependency components, source-copied | ✓ Good — Tailwind 4 + @tailwindcss/vite plugin worked; Radix portals needed jsdom mocking |
| tauri::async_runtime::spawn (not tokio::spawn) | tokio::spawn panics on Windows in Tauri 2 event listeners | ✓ Good — confirmed by Tauri issue #10289 |
| ack-before-decode for consume_message (D-10) | Avoids re-delivery on decode error; matches typical fire-and-forget dev tool usage | ✓ Good — simpler Rust code; acceptable for dev tool context |

**Issues Resolved During Milestone:**

- AMQP basic_publish ShortString type required .into() on &str args (not obvious from lapin docs)
- reqwest 0.13 renamed feature from 'rustls-tls' to 'rustls' (breaking change not in changelog)
- zod v4 incompatibility with @hookform/resolvers — pinned to ^3.24.2
- shadcn nova preset used (not zinc — CLI removed --preset=zinc in newer versions)
- Radix UI portal/pointer events incompatible with jsdom — mocked with native `<select>` in tests
- vi.hoisted() required for Vitest module-scope mocks with hoisted vi.mock() factories
- macOS sandbox required disabling in Entitlements.plist for arbitrary file read as dev tool
- lapin Connection::close() takes ShortString not &str — "".into() for string literals
- ConsumeResult fields need #[serde(rename_all = "camelCase")] — Tauri does NOT auto-convert return value field names

**Issues Deferred:**

- Linux keychain (libsecret) install documentation for distribution — libsecret dependency not bundled
- Bytes field (base64 input) — FORM-V2-01 deferred to v2
- Map field rendering (map<K, V>) — FORM-V2-02 deferred to v2

**Technical Debt Incurred:**

- REQUIREMENTS.md traceability table not updated incrementally during development — was stale at milestone close
- No CI/CD pipeline configured — builds are manual
- No app distribution pipeline (notarization, signing) — distributable binary not yet packaged for team
