<objective>
Close two BLOCKER gaps from phase 22 VERIFICATION.md that prevent the phase goal from being achieved.

CR-01: `StepResultIpc` in ipc.ts declares `step_id` (snake_case), but the Rust `StepResult` struct applies `#[serde(rename_all = "camelCase")]` (plan_runner.rs:97), which serializes the field as `stepId` over the Tauri IPC channel. The TypeScript side reads `ipc.step_id` which is always `undefined` at runtime.

CR-02: The break condition in the step execution loop in usePlanRunner.ts is `if (stopOnError && !isCancelling) break`. When the user clicks Stop, `isCancelling` becomes `true`, making `!isCancelling` false — the break never fires and all remaining steps execute normally as if Stop was never clicked.

Purpose: Both fixes are surgical (3–5 lines total). No new files, no architectural changes.
Output: src/lib/ipc.ts and src/hooks/usePlanRunner.ts with corrected field name and break condition.
</objective>
