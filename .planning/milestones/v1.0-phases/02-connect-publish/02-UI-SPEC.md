## Spacing Scale

Declared values (multiples of 4 only). Source: Tailwind 4 utility classes inferred from Sidebar.tsx (`p-4 gap-4 gap-2`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot margins, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing, between adjacent form fields (`gap-2`, `p-2`) |
| md | 16px | Default element spacing, sidebar padding, button padding (`gap-4`, `p-4`) |
| lg | 24px | Section padding, modal content padding (`gap-6`, `p-6`) |
| xl | 32px | Layout gaps, publish bar horizontal padding (`gap-8`, `p-8`) |
| 2xl | 48px | Major section breaks | 
| 3xl | 64px | Page-level spacing |

Exceptions:
- Status indicator dots: 8px × 8px circle (`w-2 h-2`) — intentionally below the xs token, not a layout unit
- Sidebar width: 240px fixed — matches Phase 1 D-02; not a spacing token

## Typography

Source: Sidebar.tsx established pattern (`text-xs`, `text-sm`, `text-lg font-semibold`, `font-medium`).

Weights: exactly 2 — 400 regular and 600 semibold. The "medium" (500) weight is dropped; the Label role is elevated to semibold given its higher visual priority over Body copy.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 18px (`text-lg`) | 600 semibold (`font-semibold`) | 1.2 | Section headings: "Proto Sender" app title, modal titles |
| Label | 14px (`text-sm`) | 600 semibold (`font-semibold`) | 1.4 | Form section labels, publish bar labels ("Queue", "Exchange", "Routing Key") |
| Body | 14px (`text-sm`) | 400 regular | 1.5 | Form field values, dropdown options, profile list items |
| Hint | 12px (`text-xs`) | 400 regular | 1.4 | Muted helper text: "Add connection", "Connect to a RabbitMQ profile to send.", status messages |

Font: Geist Variable (variable font — renders 400/600 without separate files).

## Color

Source: src/index.css CSS variables (radix-nova neutral preset). Both light and dark modes are defined.

| Role | CSS Variable | Light Value | Dark Value | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` | `oklch(0.145 0 0)` | App window background, main content area |
| Secondary (30%) | `--sidebar` / `--card` | `oklch(0.985 0 0)` | `oklch(0.205 0 0)` | Sidebar surface, modal surfaces, publish bar background |
| Accent (10%) | `--primary` | `oklch(0.205 0 0)` | `oklch(0.922 0 0)` | Send button fill, active profile indicator, form submission button |
| Muted | `--muted-foreground` | `oklch(0.556 0 0)` | `oklch(0.708 0 0)` | Hint text, "Add connection" text, placeholder text |

- Send button (primary action, publish bar)
- Active profile highlight in profile management modal

|-------|--------------|-------|-------|
| Connected | `bg-emerald-500` | green | Status dot (D-02), sidebar connection section |
| Error | `bg-destructive` | red (`--destructive`) | Status dot (D-02), sidebar connection section |
| Disconnected | `bg-muted-foreground` | neutral gray | Status dot (D-02), sidebar connection section |

## Component Inventory

### Connection Section (Sidebar — replaces `<div className="flex-1" />`)

**Component:** `src/components/sidebar/ConnectionSection.tsx`

Layout (vertical stack, `flex flex-col gap-2`, inside `p-4`):
1. Section label: `text-sm font-semibold` — "Connection"
2. If no profiles saved: muted hint `text-xs text-muted-foreground` — "Add connection" (link/button style, matches "Load a .proto file to get started" pattern)
3. If profiles exist:
   - shadcn Select dropdown (same structure as Message Type selector): profile name as value
   - Row below dropdown: `flex items-center gap-2` — [status dot 8×8px circle] [status text `text-xs text-muted-foreground`] [Manage button with gear icon, `variant="ghost"` `size="icon"` `aria-label="Manage connection profiles"`]

Status dot: `w-2 h-2 rounded-full` with color class from status indicator table above.

### Profile Management Modal

**Component:** `src/components/connection/ProfileManagementModal.tsx`

Pattern: mirrors `IncludePathDialog.tsx` — shadcn Dialog with `max-w-lg`.

Layout:
1. DialogHeader: "Connection Profiles" title
2. Saved profile list: each row shows `[profile name text-sm] [Connect button size-sm] [Delete button size-sm variant="ghost" text-destructive]`
3. Separator
4. "+ New Profile" button — `variant="outline"` — expands inline form below when clicked
5. Inline form (visible when adding): 6 always-visible fields (D-08), test result row, action buttons

**Profile form fields (always all 6, no collapsing — D-08):**

| Field | Input type | Default | Placeholder |
|-------|-----------|---------|-------------|
| Profile Name | text | — | "e.g. Local RabbitMQ" |
| Host | text | — | "localhost" |
| Port | number | 5672 | "5672" |
| Virtual Host | text | "/" | "/" |
| Username | text | — | "guest" |
| Password | password (masked) | — | "••••••••" |
| Management API Port | number | 15672 | "15672" |

**Form field labels** use `text-sm font-semibold` (Label role — 600 semibold).

**Connection test result row (D-06):** below the 6 fields, `flex items-center gap-2 text-sm`:
- Loading: `<Loader2 className="animate-spin w-4 h-4" />` + "Testing connection…"
- Success: `<CheckCircle2 className="text-emerald-500 w-4 h-4" />` + "Connected" (`text-emerald-500`)
- Failure: `<XCircle className="text-destructive w-4 h-4" />` + error message text (`text-destructive`)

Form action buttons: `[Cancel button variant="outline"] [Save & Connect button variant="default"]` — right-aligned, `flex justify-end gap-2`. Save triggers test then activate (D-07).

**Delete profile confirmation:** shadcn AlertDialog — triggered by Delete button in profile list.
- AlertDialogTitle: "Delete Profile"
- AlertDialogDescription: "Delete [profile name]? This will also remove the saved password from your OS keychain. This cannot be undone."
- AlertDialogCancel: "Keep Profile"
- AlertDialogAction: "Delete Profile" (destructive styling)

### Publish Bar

**Component:** `src/components/publish/PublishBar.tsx`

**Visual hierarchy focal point:** The Send button is the sole accent-colored (`bg-primary`) interactive element in the publish bar and serves as the primary focal point of the entire bar.

Position: persistent bar at top of main panel, above `<FormPanel>`. Background: `bg-card border-b border-border`. Padding: `px-4 py-2` (16px horizontal / 8px vertical).

Layout: `flex items-center gap-4 flex-wrap`:

1. **Mode toggle (D-10):** shadcn RadioGroup with horizontal layout (`flex gap-1`), styled as segmented control.
   - Two RadioGroupItem values: "queue" | "exchange"
   - Default: "queue" (D-10 specifics)
   - Visual: each item rendered as a Button-styled label (`variant="outline"`, active state uses `bg-primary text-primary-foreground`)
   - Toggle labels use `text-sm font-semibold` (Label role)

2. **Target picker (D-09, D-11, D-03-PUBL):**
   - Queue mode: shadcn Select showing fetched queue names OR plain Input if Management API unavailable
   - Exchange mode: shadcn Select showing filtered exchange names OR plain Input if unavailable
   - Management API badge (D-11): `<Badge variant="outline" className="text-xs gap-1">` with status dot + "Live" or "Manual" text
   - Badge placement: to the right of the picker, `flex items-center gap-2`

3. **Routing key (D-10 — Exchange mode only):** shadcn Input, `placeholder="Routing key"`, `w-48`. Hidden when Queue mode active. Label: `text-sm font-semibold` (Label role).

4. **Send button (D-12):**
   - Enabled: `<Button variant="default">Send</Button>`
   - Disabled when no active connection: `<Button disabled>Send</Button>` wrapped in `<TooltipProvider><Tooltip><TooltipTrigger>…</TooltipTrigger><TooltipContent>Connect to a RabbitMQ profile to send.</TooltipContent></Tooltip></TooltipProvider>`
   - Icon: `<Send className="w-4 h-4 mr-2" />` (Lucide `Send` icon) prefixed to label

### Toast Notifications (Sonner)

Placement: top-right (default Sonner position). Non-blocking.

| Event | Variant | Duration | Message |
|-------|---------|---------|---------|
| Successful send (D-13) | default | 3000ms | "Message sent to [queue or exchange name]" |
| Failed send (D-14) | destructive / error | 5000ms | "Send failed: [error message from backend]" |

Implementation: `toast("Message sent to " + target)` and `toast.error("Send failed: " + error)` from `sonner`. Add `<Toaster />` to `App.tsx`.

## Interaction States

### Connection Status Dot (D-02)

| State | Visual | When |
|-------|--------|------|
| Not connected | Gray dot (`bg-muted-foreground`) | No profiles, or no active profile |
| Connected | Green dot (`bg-emerald-500`) | Profile saved+tested successfully, or switched and re-tested |
| Error | Red dot (`bg-destructive`) | Test failed on save, or publish failed with AMQP error |

Transitions are user-action-driven only — no background polling.

### Management API Badge (D-11)

| State | Badge | Picker |
|-------|-------|--------|
| Live | green dot + "Live" text | shadcn Select with fetched names |
| Manual | amber dot + "Manual" text | Plain text Input |

Fallback is silent — no error dialog. The badge communicates the mode change.

### Send Button (D-12)

| State | Visual | Behavior |
|-------|--------|---------|
| No active connection | `disabled` prop, grayed | Tooltip on hover: "Connect to a RabbitMQ profile to send." |
| Active connection, no target selected | `disabled` prop | No tooltip (state is self-evident from empty picker) |
| Active connection, target selected | enabled, `variant="default"` | Calls `publish_message` IPC |
| Sending in progress | disabled + `<Loader2 className="animate-spin" />` icon | Prevents double-submit |

### Profile Form (D-06, D-07, D-08)

| State | Visual |
|-------|--------|
| Empty form (before Save) | All 6 fields visible, no test result row |
| Saving (test in progress) | Save button disabled, spinner in test result row |
| Test passed | Green checkmark + "Connected" in test result row; modal can be closed |
| Test failed | Red X + error text in test result row; form remains open, all fields editable |
