## Interaction Contract

The combobox trigger (a button styled to match `<Input className="w-48">`) is the primary visual anchor of the routing key section — it is the element users see first and interact with to access suggestions.

### State Machine: Routing Key Widget

```
selectedExchange = ""           → render nothing (mode="queue" hides the section)
selectedExchange set            → fetch bindings (D-04: show Loader2 spinner)
  ↓ fetch success, bindings []  → show combobox in open state with empty message "No bindings found"
  ↓ fetch success, bindings […] → show combobox with suggestion list
  ↓ API unreachable (D-10)      → silently fall back to plain <Input> (no error state)
  ↓ exchange type = headers/fanout (D-03) → render plain <Input> + hint text (D-06)
```

State determination order (checked top to bottom):

1. If exchange type is `headers` or `fanout` → plain Input + hint text. Done.
2. If Management API is unreachable (`managementStatus !== "live"`) → plain Input, no hint. Done.
3. If bindings fetch in progress → combobox trigger with Loader2 in place of ChevronDown icon.
4. If bindings fetched, list may be empty or populated → combobox (popover + command list).

### Combobox Widget (D-01, D-02)

**Trigger:** A button styled to match `<Input className="w-48">`. Width: `w-48`. Height: `h-9` (matches shadcn Input default). Border: `border border-input rounded-md`. Background: `bg-background`. Text alignment: left-aligned placeholder text.

**Trigger content (loading):** `<Loader2 className="ml-auto h-4 w-4 animate-spin text-muted-foreground" />` positioned at `ml-auto` (right side of trigger). Free-type remains enabled during loading.

**Trigger content (loaded):** Current routing key value (or placeholder "Routing key") on the left; `<ChevronsUpDown className="ml-auto h-4 w-4 shrink-0 text-muted-foreground" />` on the right.

**Popover content:** `<PopoverContent className="w-48 p-0">` — matches trigger width exactly. No extra padding; Command handles internal spacing.

**Suggestion list (Command):** `<Command>`. Inner: `<CommandInput>` for filtering (placeholder: "Filter keys…"), `<CommandList>` → `<CommandEmpty>` + `<CommandGroup>`.

**Empty state inside combobox (fetch returned []):** `<CommandEmpty>` renders "No bindings found". Free-type still works — user can type any key in `<CommandInput>` and press Enter or click away to confirm.

**Free-type behavior (D-02):** The `<CommandInput>` value is synced to `routingKey` state on every keystroke. User is never forced to pick from the list.

**Wildcard pattern items (D-07, D-08, D-09):** A suggestion item containing `*` or `#` renders:
```
<CommandItem>
  <span className="flex-1 truncate">{bindingKey}</span>
  <Badge className="ml-2 bg-amber-100 text-amber-900 dark:bg-amber-900/30 dark:text-amber-300 font-semibold shrink-0">
    pattern
  </Badge>
</CommandItem>
```
Selection copies `bindingKey` as-is into `routingKey` state (D-09). No additional inline warning on selection.

**Exact key items:** `<CommandItem>{bindingKey}</CommandItem>` — no badge.

**Selected state:** When `routingKey` matches a suggestion, that item shows a `<Check className="ml-auto h-4 w-4" />` icon (standard shadcn combobox pattern).

### Loading Spinner (D-04)

While bindings fetch is in progress, the combobox trigger renders with a `<Loader2>` instead of the chevron:
- Size: `h-4 w-4`
- Class: `animate-spin text-muted-foreground`
- Position: `ml-auto` inside the trigger flex row
- The trigger remains interactive (user can click to open); the popover shows `<CommandEmpty>Loading…</CommandEmpty>` if opened during fetch

### Plain Input Fallback (D-03, D-10)

When falling back to plain `<Input>`:
```
<Input
  placeholder="Routing key"
  className="w-48"
  value={routingKey}
  onChange={(e) => setRoutingKey(e.target.value)}
/>
```
No visual error state, no spinner — identical to current implementation at line 312–317 of PublishBar.tsx.

### Exchange Type Badge in Dropdown (D-05)

Each `<SelectItem>` in the exchange dropdown renders as a flex row:
```
<SelectItem key={ex.name} value={ex.name}>
  <span className="flex items-center gap-2">
    {ex.name}
    <Badge variant="outline" className="text-xs text-muted-foreground font-semibold">
      [{ex.exchange_type}]
    </Badge>
  </span>
</SelectItem>
```
Badge text format: `[topic]`, `[direct]`, `[fanout]`, `[headers]` — lowercase, bracketed.

### Hint Text (D-06)

Placed immediately below the routing key input as a `<p>` tag in the same flex-column wrapper.

**Fanout exchange:**
> "Routing key is ignored for fanout exchanges."

**Headers exchange:**
> "Headers exchanges route by message headers, not routing key."

Styling:
```
className="text-xs text-muted-foreground mt-1"
```

No icon. No tooltip. Plain text only. Weight: 400 (font-normal) — inherits from parent, no weight class applied.

### Routing Key Section Layout

```
<div className="flex flex-col gap-0">
  <div className="flex items-center gap-2">
    <label className="text-sm font-semibold">Routing Key</label>
    {/* combobox or Input rendered here */}
  </div>
  {/* Hint text rendered conditionally here — only for headers/fanout */}
  {isHintExchange && (
    <p className="text-xs text-muted-foreground mt-1">{hintText}</p>
  )}
</div>
```

The outer wrapper changes from `flex items-center gap-2` to `flex flex-col gap-0` to accommodate the hint text below. The label + input row remains `flex items-center gap-2`.
