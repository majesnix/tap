Exceptions: 12px (`sm+`) is included in the scale because the existing `FormPanel.tsx` header uses `py-3` — this phase must match that established rhythm rather than deviate from it.

## Layout Contract

### Panel Structure (D-01, D-02)

When the block library is **open**:
```
<main> flex row
  ├── <BlockLibraryPanel>  w-64 (256px fixed), flex-col, border-r border-border
  └── <FormPanel>          flex-1
```

When the block library is **closed**:
```
<main> flex col (current structure unchanged)
  └── <FormPanel>          flex-1
```

The `BlockLibraryPanel` is **conditionally rendered** (not mounted when closed). State is local React `useState` in `AppLayout` (or hoisted to FormPanel if toggle button lives there). This is a session-only UI state — not persisted.

### Layout Constraint Note

With panel open: `288px (left sidebar) + 256px (block panel) + 320px (right panel) = 864px fixed`. On a 1280px display, `FormPanel` receives ~416px of space. This is locked per D-02 — the executor must not further constrain `FormPanel` width. The toggle is default-off so most sessions are unaffected.

### Toggle Button Placement (D-03)

FormPanel header right side, left of the existing Braces (JSON toggle) button:
```
[Library icon button] [Braces icon button]
```
Both use `variant="ghost" size="icon-sm"`.

## BlockLibraryPanel: Two-View Structure (D-05)

### View 1 — List View (default)

**Primary focal point: block name list** (the `<span class="text-sm truncate flex-1">` within each row — this is the primary scannable content the user reads to identify and select blocks).

```
BlockLibraryPanel (h-full flex-col border-r border-border)
  ├── Header (px-4 py-3 border-b border-border shrink-0 flex items-center justify-between)
  │     ├── <h2 text-sm font-semibold> "Block Library"
  │     └── <Button variant="ghost" size="icon-sm" aria-label="New block"> <Plus />
  └── ScrollArea (flex-1 min-h-0)
        ├── [empty state — only when blocksLoaded && blocks.length === 0]
        │     centered text-sm text-muted-foreground
        │     heading: "No blocks yet"
        │     body: "Save JSON snippets you can reuse across messages."
        └── [block list rows — one per block]
              each row: px-3 py-2 flex items-center justify-between hover:bg-muted rounded-sm
                ├── <span class="text-sm truncate flex-1">  {block.name}
                └── <div class="flex items-center gap-2 shrink-0">
                      <Button variant="ghost" size="icon-sm" aria-label="Edit {name}"> <Pencil size={14} />
                      <Button variant="ghost" size="icon-sm" aria-label="Delete {name}"> <Trash2 size={14} />
```

**Hydration gate:** While `blocksLoaded === false`, render nothing in the scroll area (no empty state, no list). Prevents flash of empty state on app start.

### View 2 — Editor View (for New + Edit)

**Primary focal point: Save block button** (`w-full mt-auto variant="default"` — the full-width placement at the bottom of the editor draws the eye as the terminal action after filling name + JSON content).

```
BlockLibraryPanel (h-full flex-col border-r border-border)
  ├── Header (px-4 py-3 border-b border-border shrink-0 flex items-center gap-2)
  │     ├── <Button variant="ghost" size="icon-sm" aria-label="Back"> <ArrowLeft />
  │     └── <h2 text-sm font-semibold> "New block" | "Edit block"
  └── Editor body (flex-1 flex-col p-4 gap-3 min-h-0)
        ├── <Input placeholder="Block name" />  (controlled, local draft state)
        ├── <CodeMirror> (flex-1, exact JsonEditor.tsx pattern — value, onChange, resolvedTheme, json() extension)
        ├── [inline error banner — same structure as JsonEditor.tsx error banner]
        │     shown for: name-empty OR invalid-json OR non-object-json
        └── <Button variant="default" class="w-full mt-auto"> Save block
```

**Save button behavior:** Always enabled. Validates on click. On failure: show inline error, stay in editor view. On success: commit to store, return to list view.

**Cancel/Back behavior:** Discards all unsaved draft state (name + JSON content). Returns to list view. No confirmation needed for unsaved changes (consistent with D-08 / AmqpPropertiesSheet pattern).

## Interaction States

| State | Component | Behavior |
|-------|-----------|----------|
| Panel closed (default) | AppLayout | FormPanel is `flex-1` filling center column |
| Panel open | AppLayout | `<main>` is flex row; BlockLibraryPanel 256px + FormPanel flex-1 |
| Toggle active | Library icon button | `bg-muted text-foreground` (matches Braces button active pattern in FormPanel) |
| List hydrating | ScrollArea area | Empty — no content rendered until `blocksLoaded === true` |
| List empty (loaded) | ScrollArea area | Centered empty state heading + body |
| Editor — new block | Editor view | Name field empty, CodeMirror pre-filled with `{}` |
| Editor — existing block | Editor view | Name field pre-filled, CodeMirror pre-filled with saved content |
| Save — name empty | Save button | Inline error banner: "Name is required"; stay in editor |
| Save — invalid JSON | Save button | Inline error banner: "Invalid JSON" + parser message; stay in editor |
| Save — non-object JSON | Save button | Inline error banner: "JSON must be an object"; stay in editor |
| Save — valid | Save button | Commit to useBlockStore, return to list view |
| Delete — trigger | Trash2 button | Open AlertDialog with block name in title |
| Delete — confirm | Delete block button in AlertDialog | Remove from store, close dialog, stay in list view |
| Delete — cancel | Keep block button in AlertDialog | Close dialog, no change |
