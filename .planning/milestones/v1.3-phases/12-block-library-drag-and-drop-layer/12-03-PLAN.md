## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| block.id → dataTransfer payload | Block ID (UUID string) is set into the browser DnD dataTransfer. It is a read-only lookup key on the drop side — not executed, not rendered as HTML. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Tampering | dataTransfer.setData('blockId', block.id) | accept | Block ID is a UUID from the in-memory Zustand store. Only the ID is put in the payload (D-02) — not the content. Drop handler does a bounds-checked store lookup. No injection surface. |
| T-12-03-02 | Information Disclosure | draggable="true" on block rows | accept | Block names are visible in the UI already. Adding draggable does not expose new information. |
| T-12-03-03 | Elevation of Privilege | Native DnD drag from BlockLibraryPanel to external targets | accept | Desktop Tauri app — user controls the environment. Dragging to a non-app target produces no effect (no drop handler in external context for `blockId` data). |
</threat_model>
