## Milestone Summary

**Key Decisions:**

- Repository made public: required for unauthenticated latest.json fetch by tauri-plugin-updater; private repo causes silent 404 on startup check
- cs.allow-unsigned-executable-memory restored: WKWebView JIT requirement under macOS Hardened Runtime; app crashes at launch without it even after notarization passes
- Version mislabelling discovered and fixed: releases v1.5.1–v1.5.6 all built binaries labelled 1.5.0 (version never bumped in Cargo.toml, tauri.conf.json, package.json after Phase 16); corrected by bumping all three to 1.5.7
- runUpdateCheck({ manual }) extraction: startup auto-check stays silent on failure; manual trigger shows error toast and "up to date" toast — first observable feedback to user on update check failure
- macOS native menu pattern: menu built in setup() with #[cfg(target_os = "macos")], emits "check-for-updates" Tauri event; UpdateChecker listens; sidebar button gated on !isMac via navigator.userAgent

**Issues Resolved:**

- Silent update check failure: .catch(console.error) swallowed all errors including 404 from private repo — fixed by making repo public + adding visible error toast on manual trigger
- Phase 17/18 checkboxes unchecked in ROADMAP during execution (tracking artifact drift) — corrected at milestone close
- REQUIREMENTS.md traceability table all "Pending" during execution — corrected at milestone close

**Issues Deferred:**

- Phase 13 live-broker UAT: 8 items require running RabbitMQ instance (ack-before-decode, AMQP metadata accuracy, multi-type decode, queue depth refresh, accordion UX, RightPanel auto-switch, FIFO-500 cap, partial-error toast) — carried from v1.4
- Windows distribution: no code signing strategy defined
- Update download progress bar: UX polish, not required for functionality
- Update delta / rollback: tauri-plugin-updater ships full binaries only

**Technical Debt Incurred:**

- Update check requires public GitHub repo; no solution for teams that want to keep the repo private
- No automated test for the full auto-update flow (requires two live releases)
