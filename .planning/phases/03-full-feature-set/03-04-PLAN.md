Filter bar with two controls:
1. Status filter: RadioGroup (shadcn) with options "all", "sent", "failed"
2. Type search: Input with placeholder "Filter by type…"

The component is a controlled UI element — it receives callbacks, does NOT own filtered state:

```typescript
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";

type StatusFilter = "all" | "sent" | "failed";

interface HistoryFilterBarProps {
  statusFilter: StatusFilter;
  typeFilter: string;
  onStatusChange: (status: StatusFilter) => void;
  onTypeChange: (query: string) => void;
}

export function HistoryFilterBar({
  statusFilter, typeFilter, onStatusChange, onTypeChange,
}: HistoryFilterBarProps) {
  return (
    <div className="flex items-center gap-3 px-3 py-2 border-b border-border">
      <RadioGroup
        value={statusFilter}
        onValueChange={(v) => onStatusChange(v as StatusFilter)}
        className="flex gap-2"
      >
        {(["all", "sent", "failed"] as const).map((s) => (
          <div key={s} className="flex items-center gap-1">
            <RadioGroupItem value={s} id={`filter-${s}`} />
            <Label htmlFor={`filter-${s}`} className="text-xs capitalize cursor-pointer">
              {s}
            </Label>
          </div>
        ))}
      </RadioGroup>
      <Input
        placeholder="Filter by type…"
        value={typeFilter}
        onChange={(e) => onTypeChange(e.target.value)}
        className="h-7 text-xs w-36"
      />
    </div>
  );
}
```

   ```typescript
   const handleResend = async (entry: HistoryEntry) => {
     const { activeProfileName } = useConnectionStore.getState();
     if (!activeProfileName) {
       toast.error("Resend failed: No active connection profile.");
       return;
     }
     try {
       await publishMessage(
         activeProfileName,
         entry.exchange,
         entry.routingKey,
         entry.payloadBytes,
       );
       toast(`Resent to ${entry.exchange || entry.routingKey}`, { duration: 3000 });
       // Append a new history entry for the resend
       await useHistoryStore.getState().appendEntry({
         id: crypto.randomUUID(),
         timestamp: new Date().toISOString(),
         messageTypeName: entry.messageTypeName,
         exchange: entry.exchange,
         routingKey: entry.routingKey,
         status: "sent",
         fieldValues: entry.fieldValues,
         payloadBytes: entry.payloadBytes,
       });
     } catch (err: unknown) {
       const message = err instanceof Error ? err.message : String(err);
       toast.error(`Resend failed: ${message}`, { duration: 5000 });
     }
   };
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| history entry → form.reset | fieldValues from history.json are deserialized and fed into react-hook-form |
| history entry → publishMessage IPC | payloadBytes from history.json are passed directly to Rust publish command |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | handleReplay — fieldValues into form.reset | accept | fieldValues originates from user's own prior form submission stored locally; no external input; user is tamping their own data |
| T-03-04-02 | Tampering | handleResend — payloadBytes direct publish | accept | payloadBytes was encoded by the app's own Rust encoder and stored locally; no external modification vector in single-user desktop app |
| T-03-04-03 | Spoofing | handleResend target (exchange/routingKey from entry) | accept | Exchange and routingKey from history were user-selected in a prior session; no spoofing vector without physical access to the machine |
| T-03-04-04 | Denial of Service | HistoryFilterBar type search — unconstrained substring match | accept | Maximum 100 entries in history; O(n) string search on 100 items is negligible; no debounce needed |
| T-03-04-05 | Elevation of Privilege | handleReplay — setActiveIndex to arbitrary tabIndex | mitigate | tabIndex is derived from openFiles.findIndex() result; -1 check guards against tab switching to invalid index; the guard already implemented in handleReplay above |
</threat_model>
