## Color

Tokens from `src/index.css` (oklch dual-mode, shadcn nova preset):

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` | App background, PlanView surface, PlanDetailPanel background |
| Secondary (30%) | `--sidebar` / `--muted` | PlanListPanel background, hover state for unselected rows, AlertDialog overlay |
| Accent (10%) | `--accent` / `--accent-foreground` | Selected plan row highlight, Plans nav button active state, "New Plan" primary CTA |
| Destructive | `--destructive` | "Delete plan" confirm button in AlertDialog only |

**60/30/10 distribution:** Dominant `--background` covers the full `PlanView` surface and detail pane. Secondary `--sidebar`/`--muted` covers the left `PlanListPanel` background and row hover states. Accent is reserved to three specific elements — not applied to all interactive elements.

Accent (`--accent`) is reserved for:
1. Selected plan row highlight (`bg-accent text-accent-foreground`)
2. Plans nav button active state when `viewMode === "plans"`
3. "New Plan" primary CTA button

Destructive is reserved for: "Delete plan" confirm button in the AlertDialog only.

**Primary visual anchor:** The selected plan row (highlighted in `--accent`) is the primary focal point inside `PlanListPanel`.

### 4. Inline Create Row (bottom of list)

Triggered by "New Plan" CTA. A new row appears at the bottom of the plan list (D-07):

```
┌──────────────────────────────────────┐
│  [  Untitled Plan (selected text)  ] │
└──────────────────────────────────────┘
```

- `<input type="text" autoFocus defaultValue="Untitled Plan" />` — all text selected on mount (`inputRef.current.select()`)
- `onKeyDown`: Enter → `commitCreate()`, Escape → `cancelCreate()`
- `onBlur`: treated as Enter (commit)
- Empty-name guard: if value is blank after trim, treat as Escape (D-09)
- Commit calls `usePlanStore.createPlan(name)`

### 5. Inline Rename (in-row)

Triggered by Rename from kebab DropdownMenu (D-08):

- Replaces plan name `<span>` with `<input type="text" autoFocus defaultValue={plan.name} />`
- Same keyboard contract as inline create: Enter commits, Escape discards
- `onBlur`: treated as Enter (commit)
- Empty-name guard: blank → discard (D-09)
- Commit calls `usePlanStore.renamePlan(id, name)`

### 6. Delete AlertDialog

Triggered by Delete from kebab DropdownMenu (D-05):

```
Delete "My Plan"?

This action cannot be undone.

[Keep plan]    [Delete plan]
```

| Element | Spec |
|---------|------|
| Component | shadcn `AlertDialog` (existing in app) |
| Title | `Delete "{plan.name}"?` |
| Description | `"This action cannot be undone."` |
| Cancel button | `"Keep plan"` — `<AlertDialogCancel>` |
| Confirm button | `"Delete plan"` — `<AlertDialogAction className="bg-destructive text-destructive-foreground hover:bg-destructive/90">` |
| On confirm | `usePlanStore.deletePlan(id)` → if `selectedPlanId === id`, reset to `null` (D-13) |

No undo/toast-undo — AlertDialog is the only safety net (D-06).

## Interaction Contracts

### View Switching

- `viewMode: "main" | "plans"` is local `useState` in `App.tsx` — NOT in any Zustand store (D-10)
- Prop-drill path: `App.tsx` → `AppLayout.tsx` (passes to `Sidebar`) → `Sidebar.tsx` (renders Plans nav button)
- `usePlanStore.loadPlans()` called in `App.tsx` `useEffect` at mount — plan data available immediately on view switch (D-11)

### Plan Selection

- `selectedPlanId: string | null` is local `useState` in `PlanView.tsx` — NOT in `usePlanStore` (D-12)
- When currently selected plan is deleted → `setSelectedPlanId(null)` (D-13)
- Clicking plan row body → `setSelectedPlanId(plan.id)`
- Clicking kebab → opens DropdownMenu without changing selection

### Keyboard Nav (inline edit)

| Key | Create row | Rename |
|-----|-----------|--------|
| Enter | Commit + `createPlan(name)` | Commit + `renamePlan(id, name)` |
| Escape | Cancel + remove pending row | Discard, restore name span |
| Blur | Same as Enter | Same as Enter |
| Empty + Enter | Cancel (D-09) | Discard (D-09) |

## Accessibility

- Plans nav button: includes text label ("Plans") — not icon-only
- Kebab button: `aria-label="Plan options"` — icon-only button requires explicit label
- Inline edit input: `autoFocus` on mount; `aria-label="Plan name"` on `<input>`
- AlertDialog: Radix `AlertDialog` provides accessible modal + focus trap by default
- Empty state icons: `aria-hidden="true"` (decorative)
- Color contrast: accent/background contrast verified by shadcn nova preset — no overrides needed
