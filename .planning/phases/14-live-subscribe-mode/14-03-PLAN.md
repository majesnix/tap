<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SubscribePanel → startSubscribe IPC | profileName, queueName, decodeTypes from UI state passed to Rust |
| Channel callback → appendMessages | DrainResult from Rust arrives asynchronously; malformed data could cause runtime errors |
| Auto-stop useEffect | Reads connectionStatus from store — store is the single source of truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-11 | Information Disclosure | Error message in status badge | mitigate | handleStart catch: only forward sanitized message — never forward raw error.message if it may contain AMQP URI/password; use generic "Subscribe failed" as fallback (ASVS V3.5) |
| T-14-12 | Denial of Service | Channel callback flooding React render cycle | mitigate | appendMessages enforces FIFO-500; Rust side enforces basic_qos(20) prefetch — double-bounded |
| T-14-13 | Elevation of Privilege | Mode toggle bypass while Running | mitigate | isModeLocked disables ToggleGroup when Running or Stopping (D-06); UI-layer enforcement |
| T-14-14 | Tampering | Auto-stop condition | accept | useEffect deps are read-only store values; no external input; low risk |
| T-14-15 | Denial of Service | Start called before Stop completes | mitigate | D-08 guard in Rust (Plan 01) returns Err("Already running"); UI Start button disabled when not Idle |
</threat_model>
