Consumer loop pattern (from PATTERNS.md):
```rust
tauri::async_runtime::spawn(async move {
    let _ = amqp_channel.basic_qos(20, BasicQosOptions::default()).await;
    let mut consumer = amqp_channel
        .basic_consume(&queue_name, "tap-subscriber", BasicConsumeOptions::default(), FieldTable::default())
        .await
        .expect("basic_consume failed");
    loop {
        tokio::select! {
            delivery = consumer.next() => {
                match delivery {
                    Some(Ok(delivery)) => {
                        let _ = delivery.acker.ack(BasicAckOptions::default()).await;
                        // decode + channel.send(result)
                    }
                    Some(Err(e)) => { let _ = tauri_channel.send(error_result(e)); break; }
                    None         => { let _ = tauri_channel.send(broker_closed_result()); break; }
                }
            }
            _ = token.cancelled() => {
                let _ = amqp_channel.basic_cancel("tap-subscriber", BasicCancelOptions::default()).await;
                let _ = conn.close(200, "normal shutdown").await;
                break;
            }
        }
    }
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → start_subscribe | profileName, queueName, decodeTypes are untrusted IPC parameters |
| Rust → AMQP broker | All AMQP protocol messages; broker responses are untrusted |
| AMQP delivery body | Raw protobuf bytes from broker; decode can fail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | start_subscribe profileName | mitigate | Validate profileName exists in profile store before using; reject with Err if not found (ASVS V5.1) |
| T-14-02 | Elevation of Privilege | start_subscribe (double-start) | mitigate | D-08 guard: lock Mutex, return Err("Already running") if Some — enforced in Task 2 (ASVS V3.7) |
| T-14-03 | Information Disclosure | AMQP URI / password in error messages | mitigate | Drop URI after connect; error messages return generic "connection failed" — never include the raw URI or password (ASVS V3.5) |
| T-14-04 | Denial of Service | Unbounded message delivery rate | mitigate | basic_qos(20) prefetch cap limits in-flight deliveries to 20 at once (D-12) |
| T-14-05 | Tampering | decode_types input | mitigate | Validate each decode_type string is a known type identifier before use; reject unknown values with Err |
| T-14-06 | Information Disclosure | JoinHandle panic message | accept | Task panic messages are internal to Rust and not forwarded to the frontend — low risk |
</threat_model>
