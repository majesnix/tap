<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend store → IPC | startSubscribe parameters are passed directly to Rust; validation must exist at both layers |
| Channel callback → store | Messages arriving via Channel callback update store state; malformed DrainResult could cause runtime errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-07 | Tampering | startSubscribe parameters | mitigate | Parameters validated in Rust command (Plan 01); frontend should validate non-empty profileName and queueName before calling (ASVS V5.1) |
| T-14-08 | Information Disclosure | subscribeError message in store | mitigate | Error strings set via setSubscribeStatus("Error", sanitizedMsg) — Rust must not pass AMQP URI or password in error strings; frontend displays as-is (ASVS V3.5) |
| T-14-09 | Denial of Service | Runaway channel callbacks flooding React state | mitigate | appendMessages already enforces FIFO-500 cap; Channel delivers at prefetch rate (20 max) from Plan 01 |
| T-14-10 | Tampering | ToggleGroup value from user interaction | accept | UI-only decision (Drain/Subscribe mode); no security consequence |
</threat_model>
