## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap-1), drag handle to name gap |
| sm | 8px | Compact element spacing (gap-2), button icon margins |
| md | 16px | Default element spacing (px-4), section content padding |
| lg | 24px | Section padding (py-3 px-4 = 12+16), empty state padding |
| xl | 32px | Layout gaps between major sub-panes |
| 2xl | 48px | Empty state vertical centering gap |
| 3xl | 64px | Page-level — not used in this phase |

Exceptions:
- Step list row height: `py-2 px-3` (8px vertical, 12px horizontal) — matches PlanListPanel plan rows exactly.
- Touch target minimum: 32px height for icon-only buttons (size="icon-sm"), 44px for drag handle area (invisible padding).
- Section header row: `px-4 py-3` with `border-b border-border` — matches PlanListPanel's header row.

Accent reserved for: nothing in this phase — no send/publish CTAs exist in the authoring phase. The `+` add button uses `variant="ghost"` matching the "New Plan" button in PlanListPanel.

Selection tint (`bg-accent`) is reserved for: selected step row in StepListPanel.

Destructive color is reserved for: AlertDialog "Delete step" action button only.

Color note: This app uses no vivid brand colors. All palette is achromatic neutral (shadcn `neutral` base). Dark mode is fully supported via the existing `.dark` CSS class — all components use semantic tokens, not raw hex values.

## Layout Contract

### PlanDetailPanel Sub-Split

```
PlanDetailPanel (flex-1 flex-row, min-h-0)
├── StepListPanel (w-60 / 240px fixed, border-r border-border, flex flex-col, shrink-0)
│   ├── Header (px-4 py-3, border-b border-border, flex items-center justify-between)
│   │   ├── "Steps" label (text-sm font-semibold)
│   │   └── "+" Button (variant="ghost" size="sm", gap-1) → DropdownMenu
│   ├── ScrollArea (flex-1 min-h-0)
│   │   └── StepRow × N  (or empty state)
│   └── [AlertDialog rendered at StepListPanel root, outside ScrollArea]
└── StepFieldEditor (flex-1, flex flex-col, min-h-0, overflow-hidden)
    ├── ScrollArea (flex-1)
    │   ├── Section: Proto file + message type
    │   ├── Separator
    │   ├── Section: Fields
    │   ├── Separator
    │   ├── Section: Target
    │   ├── Separator
    │   └── Section: Response mode
    └── [no footer — auto-save, no Save button]
```

Width note: CONTEXT.md D-01 specifies "~240px" for the step list (to match Phase 20 plan list proportions). PlanListPanel uses `w-72` (288px). Use `w-60` (240px) for StepListPanel as specified by D-01 — this is intentionally narrower than the plan list, reflecting that step names are typically shorter. Do not match PlanListPanel's `w-72`.

### StepRow Layout

```
StepRow (flex items-center py-2 px-3, cursor-pointer)
├── GripVertical icon (size=14, text-muted-foreground, cursor-grab, mr-1, flex-shrink-0)
├── Step name (text-sm truncate flex-1)
└── Kebab button (variant="ghost" size="icon-sm", onClick: e.stopPropagation())
    └── DropdownMenu → Rename / Duplicate / Delete
```

Drag state: row gets `opacity-50` when it is the active drag source (via `useSortable` `isDragging`). DragOverlay renders step name as a detached floating row with `bg-background border border-border shadow-md rounded`.

### StepFieldEditor Section Layout

Each section uses the pattern:
```
<div className="px-4 py-3 border-b border-border">
  <h3 className="text-sm font-semibold mb-3">{Section title}</h3>
  {/* section content */}
</div>
```

Sections in order (D-08, non-negotiable):
1. **Proto file + message type** — title: "Proto file"
2. **Fields** — title: "Fields" (renders field primitives; shows "No message type selected." helper if none)
3. **Target** — title: "Target" (queue/exchange radio toggle + name input/select + routing key)
4. **Response mode** — title: "Response mode" (no-wait / correlation-id / first-arrival radio + per-mode inputs)

Do NOT use `<Tabs>` — D-08 is explicit: scrollable sections, no tabs.

## Interaction Contracts

### Step Selection

- Clicking a step row sets it as selected (`selectedStepId` local state in PlanDetailPanel).
- Selected row: `bg-accent text-accent-foreground` (exact class, matches PlanListPanel).
- Non-selected row hover: `hover:bg-muted/50` (exact class, matches PlanListPanel).

### Inline Rename

Exact pattern from Phase 20 `InlineEditRow` — replicate without modification:
- Row switches from `StepRow` to `InlineEditRow` when `renamingId === step.id`.
- Input: `autoFocus`, `defaultValue={step.name}`, `text-sm bg-transparent border-b border-border focus:outline-none w-full`.
- `useEffect`: `inputRef.current?.select()` on mount (selects all text).
- Enter: commit non-empty value; discard if empty.
- Escape: set `cancellingRef.current = true` BEFORE calling `blur()` to avoid spurious commit.
- Blur: commit non-empty value; discard if empty (unless Escape already handled via ref guard).

### Delete Confirmation (AlertDialog)

Pattern from Phase 20 `PlanListPanel` — replicate exactly:
- AlertDialog rendered at component root (NOT inside DropdownMenuItem or ScrollArea).
- DropdownMenuItem "Delete" must call `e.preventDefault()` before setting `stepToDelete` state.
- AlertDialog `open={!!stepToDelete}` controlled by `onOpenChange`.
- After delete: if the deleted step was selected, clear `selectedStepId` to `null`.

### Auto-Save

- StepFieldEditor uses `watch()` from react-hook-form.
- Debounce: 300ms.
- On change: serialize form values to JSON string, call `usePlanStore.updateStep(planId, stepId, { field_values: json })`.
- No Save button, no dirty indicator.
- Optimistic write with rollback on error (matches all existing store mutations).

### "+" Add Menu (DropdownMenu)

- Trigger: `variant="ghost" size="sm"` button in StepListPanel header, label "Add step", `<Plus size={14} />` icon left of label.
- Three items: "Blank step" / "From history" / "From block library".
- "Blank step": creates a new step via `usePlanStore`, auto-selects it, initializes form from `buildDefaultValues`.
- "From history": opens `StepHistoryPicker` (Sheet).
- "From block library": opens `StepBlockPicker` (Sheet).

### History Picker (Sheet)

- Component: `StepHistoryPicker` as a Sheet (not Dialog). Reason: Sheet is the established picker pattern in this app (AmqpPropertiesSheet).
- Content: scrollable list of `HistoryEntry` items, newest first. Mirror `HistoryTable` layout.
- Selecting an entry: closes the Sheet, creates a new PlanStep pre-filled from the entry (per D-11).
- Empty state: "No history yet. Send a message first."
- Error fallback (no matching proto file): toast error "Open the .proto file for [messageTypeName] first, then retry."

### Block Picker (Sheet)

- Component: `StepBlockPicker` as a Sheet (same pattern as history picker).
- Content: scrollable list of blocks from `useBlockStore`. Mirror BlockLibraryPanel list.
- Selecting a block: closes Sheet, creates new PlanStep pre-filled with `field_values` from `block.content`. Proto path and message type left blank (user sets manually — D-12 accepted behavior).
- Empty state: "No blocks saved yet."

### Drag-and-Drop Reorder

- Library: `@dnd-kit/sortable` (new dependency) + `@dnd-kit/core` (already installed).
- `DndContext` wraps `StepListPanel` only — plan-scoped, separate from AppLayout's block DndContext (D-14).
- Strategy: `verticalListSortingStrategy`.
- Sensors: `PointerSensor` with activationConstraint `{ distance: 4 }` (matches AppLayout pattern, prevents accidental drag on click).
- `DragOverlay`: floating card showing dragged step name — `bg-background border border-border shadow-md rounded px-3 py-2 text-sm`.
- On `dragEnd`: call `usePlanStore.reorderSteps(planId, fromIndex, toIndex)`. Persist immediately.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Add step" (Plus icon + label on "+" dropdown trigger button) |
| Step list panel header | "Steps" |
| Empty state — no steps | Heading: "No steps yet" / Body: "Use the + button to add your first step." |
| Empty state — no selection | (no heading) Body: "Select a step to edit it." |
| Fields section — no message type | "No message type selected." (text-xs text-muted-foreground) |
| History picker empty | "No history yet. Send a message first." |
| Block picker empty | "No blocks saved yet." |
| Step delete dialog title | Delete "{step name}"? |
| Step delete dialog body | This action cannot be undone. |
| Step delete cancel | Keep step |
| Step delete confirm | Delete step |
| History import — proto mismatch | Toast error: "Open the .proto file for [messageTypeName] first, then retry." |
| Auto-save error | Toast error: "Failed to save step. Changes may be lost." |
| Duplicate step name | "{original name} (copy)" |
| Blank step default name | "Untitled Step" |

Notes:
- "Keep step" and "Delete step" mirror Phase 20's "Keep plan" / "Delete plan" — intentionally consistent.
- No "Save" label appears anywhere in Phase 21 (auto-save, D-09).
