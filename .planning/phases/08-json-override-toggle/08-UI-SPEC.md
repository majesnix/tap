## Component Specifications

### 1. Toggle Button

**Placement:** Right side of existing FormPanel header div. Change header div from:
```tsx
<div className="px-4 py-3 border-b border-border shrink-0">
```
to:
```tsx
<div className="px-4 py-3 border-b border-border shrink-0 flex items-start justify-between">
```

The left side contains the existing `h2` + `p` block. The toggle button sits on the right.

**Specification:**

| Property | Value |
|----------|-------|
| Component | `<Button>` from `src/components/ui/button.tsx` |
| Variant (inactive) | `ghost` |
| Variant (active/JSON mode) | `ghost` with `aria-pressed={true}` — visual feedback via ghost hover state |
| Size | `icon-sm` (28×28px, matches small action buttons elsewhere) |
| Icon | `Braces` from `lucide-react` (renders `{ }` glyph, semantic for JSON) |
| `aria-label` (inactive) | `"Edit as JSON"` |
| `aria-label` (active) | `"Return to form"` |
| `aria-pressed` | `{isJsonMode}` |
| `title` tooltip | Same as `aria-label` value (native browser tooltip, no extra Tooltip component needed) |

**Active state visual indicator:** In JSON mode, apply `bg-muted text-foreground` CSS override via `className` prop to make the pressed state visible without a separate variant — consistent with ghost `aria-expanded` pattern in button.tsx.

```tsx
<Button
  variant="ghost"
  size="icon-sm"
  aria-label={isJsonMode ? "Return to form" : "Edit as JSON"}
  aria-pressed={isJsonMode}
  title={isJsonMode ? "Return to form" : "Edit as JSON"}
  className={isJsonMode ? "bg-muted text-foreground" : ""}
  onClick={handleToggle}
>
  <Braces />
</Button>
```

### 2. JsonEditor Component

**File:** `src/components/form/JsonEditor.tsx`

**Container placement:** In JSON mode, `FormPanel` replaces the `<ScrollArea>` + `<ProtoFormRenderer>` block with a `flex-1 flex flex-col min-h-0` div hosting `<JsonEditor>`. CodeMirror manages its own internal scrolling — do NOT nest inside a `<ScrollArea>` (this is a refinement of D-02; nesting CM inside ScrollArea stacks two scroll containers and breaks CM gutters).

```tsx
// FormPanel JSON mode render:
<div className="flex-1 flex flex-col min-h-0">
  <JsonEditor
    value={jsonSnapshot}
    onChange={setJsonDraft}
    resolvedTheme={resolvedTheme}
  />
</div>
```

**JsonEditor layout:**
- Editor fills all available height: `className="flex-1 min-h-0"` on the CodeMirror wrapper
- Error banner rendered below editor when parse error exists (see §3)

**CodeMirror setup:**

| Property | Value |
|----------|-------|
| Package | `@uiw/react-codemirror ^4.25.9` |
| Language extension | `@codemirror/lang-json ^6.0.x` (import: `import { json } from "@codemirror/lang-json"`) |
| Theme prop | `theme={resolvedTheme === "dark" ? "dark" : "light"}` — uses `next-themes` `resolvedTheme` (same pattern as Phase 5) |
| `height` prop | `"100%"` — fills flex parent |
| `basicSetup` | Default (provides line numbers, syntax highlighting, bracket matching) |
| `extensions` | `[json()]` |
| `onChange` | `(value) => setJsonDraft(value)` — captures keystrokes as draft string; no live parse |

**Dark mode wiring:** Import `useTheme` from `"next-themes"` inside `FormPanel.tsx` and pass `resolvedTheme` as prop. Do NOT import a separate `@codemirror/theme-one-dark` — the built-in `"dark"` theme is sufficient.

### 3. Inline Error Banner

**Trigger:** Appears when user clicks "Return to form" (toggle button) and `JSON.parse(jsonDraft)` throws.

**Validation timing:** Parse only on toggle-back click — NOT on keystroke. CodeMirror's built-in JSON syntax cues provide live feedback. No debounce needed.

**Layout:** Rendered inside `JsonEditor.tsx`, below the CodeMirror editor, outside the editor's scroll region.

```
┌─────────────────────────────────────────────────────┐
│  [CodeMirror editor — flex-1, owns scroll]          │
└─────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────┐
│  △  Invalid JSON                                    │
│     Unexpected token at position 42                 │
│                                                     │
│     [Fix JSON]   [Discard changes]                  │
└─────────────────────────────────────────────────────┘
```

**Tailwind classes for banner container:**
```
className="mx-4 mt-2 mb-3 rounded-md border border-destructive/40 bg-destructive/10 p-3"
```

**Banner internal layout:**

| Element | Spec |
|---------|------|
| Icon | `<TriangleAlertIcon className="size-4 text-destructive shrink-0 mt-1" />` (from lucide-react — already imported in sonner.tsx) |
| Heading | `<span className="text-xs font-semibold text-destructive">Invalid JSON</span>` |
| Parse error body | `<p className="text-xs text-destructive mt-1" role="alert">{parseError}</p>` — verbatim from `JSON.parse()` catch `e.message` |
| Button row | `flex gap-2 mt-2` |
| "Fix JSON" button | `<Button variant="outline" size="sm">Fix JSON</Button>` |
| "Discard changes" button | `<Button variant="destructive" size="sm">Discard changes</Button>` |

**"Fix JSON" behavior:** Dismiss banner (clear `parseError` state), stay in JSON mode, focus returns to editor.

**Discard changes behavior:** Switch to form mode, call `resetRef.current(entrySnapshot)` where `entrySnapshot` is the `latestValues` captured at moment `isJsonMode` was set to `true`. User loses only the JSON edits, not pre-existing form state. No AlertDialog confirmation — loss is bounded to JSON-mode edits only (D-05, D-06).

### 4. Unknown Fields Toast

**Trigger:** After successful `JSON.parse`, during unknown-field detection (D-07, D-08).

**Call:** `toast.warning(message)` — uses `TriangleAlertIcon` already registered in `src/components/ui/sonner.tsx`.

**Format string:**
- 1 unknown field: `"1 unknown field ignored: fieldName"`
- N unknown fields: `"{N} unknown fields ignored: field1, field2, field3"`

**Behavior:** Non-blocking. Form proceeds with unknown keys stripped. Toast auto-dismisses (sonner default duration).

## State Machine: Mode Transitions

```
FORM MODE ──[click toggle]──────────────────────────────> JSON MODE
           capture latestValues as entrySnapshot             │
           set jsonSnapshot = JSON.stringify(latestValues)   │
                                                             │
JSON MODE ──[click toggle]──> parse jsonDraft               │
                               │                             │
                    ┌─── VALID ─────────────────────────────┤
                    │          strip unknown keys             │
                    │          toast.warning (if unknowns)    │
                    │          resetRef.current(cleanedValues) │
                    │          set isJsonMode = false          │
                    │                                         │
                    └─── INVALID ──> show inline error banner │
                                     stay in JSON mode        │
                                                             │
BANNER ──[Fix JSON]──> dismiss banner, stay in JSON mode    │
        ──[Discard changes]──> resetRef.current(entrySnapshot) │
                                set isJsonMode = false         │
```

## Copywriting Contract

| Element | Exact copy |
|---------|-----------|
| Toggle aria-label (inactive) | `"Edit as JSON"` |
| Toggle aria-label (active) | `"Return to form"` |
| Toggle title tooltip (inactive) | `"Edit as JSON"` |
| Toggle title tooltip (active) | `"Return to form"` |
| Error banner heading | `"Invalid JSON"` |
| Error banner body | `{e.message}` — verbatim JSON.parse error, no rewording |
| "Fix" button label | `"Fix JSON"` |
| "Discard" button label | `"Discard changes"` |
| Unknown fields toast (singular) | `"1 unknown field ignored: {fieldName}"` |
| Unknown fields toast (plural) | `"{N} unknown fields ignored: {names.join(', ')}"` |
| Empty editor (no form values yet) | `{}` — JSON.stringify of empty object; no additional copy |

No destructive confirmation dialog — "Discard changes" button in banner is the confirmation step (bounded scope: only JSON-mode edits lost, D-05 rationale).
