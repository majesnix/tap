## UI States

All six states must be handled explicitly. No implicit fallthrough.

| State | Trigger | Visual |
|-------|---------|--------|
| `idle` | Tab opened; no read attempted yet | Muted placeholder: "Select a reply queue and click Read" (`text-xs text-muted-foreground`) |
| `no-active-connection` | `connectionStatus !== "connected"` | Read button disabled; `TooltipContent`: "Connect to a RabbitMQ profile to read." |
| `loading` | IPC `consume_message` in-flight | `Loader2` spinner inside Read button; button disabled (D-16) |
| `empty` | IPC returns empty-queue sentinel | Inline text "Queue empty" replaces idle placeholder — no toast (D-03) |
| `success` | Decode succeeded | `ResponseDecodedView` (collapsible key-value tree) on top; `ResponseHexSection` (raw hex + copy button) below |
| `decode-error` | prost-reflect decode fails | Inline error: "Decode failed: [reason]. Showing raw bytes." + raw hex only; message was acked (D-10) |

## Layout Contract

### ResponseTab (3rd tab in RightPanel)

```
┌─────────────────────────────────────────┐
│  ResponseQueuePicker                    │  ← px-4 py-2 header strip
│  [Select queue ▾]  [●Live]  [Read]      │
├─────────────────────────────────────────┤
│  ResponseDecodedView  (ScrollArea)      │  ← flex-1, overflow-hidden
│  ▶ fieldName: value                     │
│    ▶ nestedField                        │
│      ▶ deeperField: value               │
├─────────────────────────────────────────┤  ← Separator
│  ResponseHexSection                     │  ← px-4 py-2
│  [Copy hex]  [Copy decoded JSON]        │
│  0a 05 68 65 6c 6c 6f …                 │  ← text-xs font-mono break-all
└─────────────────────────────────────────┘
```

- RightPanel tab order: `hex` | `history` | `response`
- Vertical stacking (not side-by-side) — matches narrow RightPanel width (D-02)
- ResponseDecodedView takes `flex-1` when result is present; collapses to zero when idle/empty/error
- ResponseHexSection always visible in `success` and `decode-error` states

## Interaction Contracts

### Read Button
- Enabled when: `connectionStatus === "connected"` AND `selectedQueue` is non-empty
- Disabled + tooltip when: `connectionStatus !== "connected"` — tooltip text: "Connect to a RabbitMQ profile to read."
- Loading: `Loader2 animate-spin w-4 h-4` inside button; button `disabled` (D-16)
- On success: replace current result with new result (D-04 — no accumulation)
- After successful read: RightPanel auto-switches to "response" tab via `lastReadAt` signal (D-12)

### ResponseQueuePicker
- Queue list loads on Response tab focus; reuses existing `fetch_queues` IPC call (D-06)
- Live mode: `Select` dropdown (same width `w-48` as PublishBar) + `bg-emerald-500` Live badge
- Manual mode: `Input` text field (`w-48`) + `bg-amber-500` Manual badge
- 401 auth error: `Badge variant="destructive"` with error text (matches PublishBar 401 pattern)
- Selected queue is session-only Zustand state; resets on app restart (D-07)

### ResponseDecodedView (collapsible key-value tree)
- Top-level fields shown expanded by default
- Nested message fields rendered as `Collapsible` nodes with `ChevronRight` (collapsed) / `ChevronDown` (expanded)
- All labels read-only — no inputs, no form controls (D-15)
- Field name: `text-sm font-semibold text-foreground`
- Field value: `text-sm text-muted-foreground` (scalar) or sub-tree (nested)
- Indentation: 16px per nesting level via `pl-4` on `CollapsibleContent`

### ResponseHexSection
- Raw hex string rendered as `<pre className="text-xs font-mono break-all whitespace-pre-wrap">`
- "Copy hex" ghost button: copies hex string, fires `toast("Hex copied", { duration: 2000 })`
- "Copy decoded JSON": copies `JSON.stringify(decoded, null, 2)`, fires `toast("JSON copied", { duration: 2000 })`
- Both copy buttons use `Copy` lucide icon + `variant="ghost" size="sm"` (matches existing ghost button pattern)

### Auto-tab-switch (D-12)
- Pattern: edge-detection ref (`prevLastReadAt`) in `RightPanel.useEffect` — identical to `lastSendAt` / History auto-switch
- Signal: `lastReadAt: number | null` in `useResponseStore`
- Transition: `null → non-null` triggers `setActiveTab("response")`
