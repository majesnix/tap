**Edit 1 — Add `if: github.event_name == 'push'` guard to "Import Apple Developer Certificate" step.**

The step currently starts (line ~41):
```yaml
      - name: Import Apple Developer Certificate
        env:
          APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
```

Add `if: github.event_name == 'push'` immediately after `- name:` and before `env:`:
```yaml
      - name: Import Apple Developer Certificate
        if: github.event_name == 'push'
        env:
          APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
          APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
          KEYCHAIN_PASSWORD: ${{ secrets.KEYCHAIN_PASSWORD }}
        run: |
          echo $APPLE_CERTIFICATE | base64 --decode > certificate.p12
          security create-keychain -p "$KEYCHAIN_PASSWORD" build.keychain
          security default-keychain -s build.keychain
          security unlock-keychain -p "$KEYCHAIN_PASSWORD" build.keychain
          security set-keychain-settings -t 3600 -u build.keychain
          security import certificate.p12 -k build.keychain -P "$APPLE_CERTIFICATE_PASSWORD" -T /usr/bin/codesign
          security set-key-partition-list -S apple-tool:,apple:,codesign: -s -k "$KEYCHAIN_PASSWORD" build.keychain
          security find-identity -v -p codesigning build.keychain
```

**Edit 2 — Add `if: github.event_name == 'push'` guard to "Verify Certificate" step.**

The step currently starts (line ~55):
```yaml
      - name: Verify Certificate
        run: |
```

Add `if: github.event_name == 'push'` immediately after `- name:`:
```yaml
      - name: Verify Certificate
        if: github.event_name == 'push'
        run: |
          CERT_INFO=$(security find-identity -v -p codesigning build.keychain | grep "Developer ID Application")
```

**Edit 4 — Insert Gatekeeper verification step after the tauri-action step.**

```yaml
          DMG=$(find src-tauri/target/universal-apple-darwin/release/bundle/dmg -name "*.dmg" | head -1)
```

  <action>
Restore the `com.apple.security.cs.allow-unsigned-executable-memory` entitlement that WR-03 (Phase 16 security review) incorrectly removed.

**Why this is being re-added:** WR-03 flagged this as "overly broad," but Apple's Hardened Runtime documentation explicitly requires it for apps embedding WKWebView. Without it, the JIT engine in WKWebView's JavaScript engine cannot write executable memory pages, and the notarized app crashes at launch with signal 9. `cs.allow-jit` alone is not sufficient — both entitlements are required. See RESEARCH.md Pitfall 1 for full analysis.

**What NOT to add:** Do NOT restore `com.apple.security.cs.allow-dyld-environment-variables`. That entitlement is genuinely optional (enables custom dyld env overrides, primarily for debugging). WR-03's concern about it was valid. Only `cs.allow-unsigned-executable-memory` is required.

Replace the current content of `src-tauri/Entitlements.plist` with exactly:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <!-- Sandbox DISABLED — required for arbitrary file read (.proto files) -->
  <key>com.apple.security.app-sandbox</key>
  <false/>

  <!-- JIT execution — required by WKWebView -->
  <key>com.apple.security.cs.allow-jit</key>
  <true/>

  <!-- Unsigned executable memory — required by WKWebView for JS JIT engine -->
  <!-- Restored: WR-03 (Phase 16) incorrectly removed this. Without it, notarized app crashes at launch. -->
  <!-- Apple Hardened Runtime docs: required for apps embedding WebKit. -->
  <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
  <true/>
</dict>
</plist>
```

This is a 3-key plist. The `app-sandbox` and `cs.allow-jit` entries are unchanged from Phase 16 final state. Only `cs.allow-unsigned-executable-memory` is added back.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub secrets → CI runner | APPLE_CERTIFICATE and other secrets cross from encrypted storage into runner env vars during signing steps |
| CI runner → Apple notary service | Unsigned zip submitted to Apple's notary API using app-specific password |
| Signed DMG → end user Mac | Gatekeeper validates notarization ticket before allowing execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Spoofing | Ad-hoc signed DMG shipped as "Developer ID signed" | mitigate | Explicit `spctl --assess` step in CI gates on actual Gatekeeper approval; build fails if DMG is not notarized |
| T-17-02 | Tampering | Compromised `tauri-apps/tauri-action` supply chain | mitigate | Action pinned to exact commit-equivalent tag `@v0.6.2` (already done in Phase 16); review release notes before bumping |
| T-17-03 | Information Disclosure | `APPLE_CERTIFICATE` secret exfiltration via CI log | mitigate | Secret is referenced via `${{ secrets.APPLE_CERTIFICATE }}` — GitHub masks secrets in logs; cert import step uses env block (not inline shell expansion) |
| T-17-04 | Information Disclosure | `certificate.p12` left on runner filesystem | accept | Runner is ephemeral (GitHub-hosted); file is destroyed after job; no persistent storage risk |
| T-17-05 | Elevation of Privilege | App-specific password reuse across services | accept | App-specific passwords are Apple-scoped; if `APPLE_PASSWORD` leaks, only notarization is at risk (not full Apple ID) |
| T-17-06 | Tampering | `cs.allow-unsigned-executable-memory` widens attack surface on Hardened Runtime | accept | Entitlement is Apple-required for WKWebView; without it app crashes; scope is limited to unsigned JIT pages (not arbitrary code execution) |
</threat_model>
