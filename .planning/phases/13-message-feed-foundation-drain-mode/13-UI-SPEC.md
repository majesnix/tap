## Interaction Contract

### Toolbar Layout (ResponseQueuePicker extension)

The toolbar is a single `flex items-center gap-2 flex-wrap` row at `px-4 py-2 border-b border-border`. Elements left to right:

1. Queue selector (Select dropdown or Input â€” existing behavior, unchanged)
2. Live / Manual / Auth-error badge (existing, unchanged)
3. Queue depth badge â€” `Badge variant="secondary"` (existing, unchanged)
4. Drain count `<input type="number">` â€” `w-12 h-9 text-sm text-center rounded-md border border-input bg-background px-1`, `min={1}`, `max={500}`, `defaultValue={10}` â€” plain HTML input, not a spinbox library
5. Drain button â€” `Button variant="default"`, label **"Drain"**, disabled when `!selectedQueue.trim() || isLoading || connectionStatus !== "connected"`

**Drain button loading state:** When `isLoading` is true, render `<Loader2 className="w-4 h-4 mr-2 animate-spin" />` inside the button before the label â€” matching the existing pattern in ResponseQueuePicker.

**Drain button disabled tooltip:** When disconnected, wrap in `TooltipProvider > Tooltip > TooltipTrigger(asChild) > span > Button` with `TooltipContent` text: `"Connect to a RabbitMQ profile to drain messages."` â€” matching existing disabled-read pattern.

**Drain count input validation:** Clamp value to [1, 500] on `blur` and before invoking drain. If value is blank or NaN on submit, default to 10.

### Feed Header (inside MessageFeedTab, above the accordion)

Single `flex items-center justify-between px-4 py-2` bar using `--muted` background.

- Left: message count label â€” `"N messages"` (14px semibold). When 0 messages: `"No messages"`.
- Right: Clear button â€” `Button variant="ghost" size="icon"` with `Trash2` lucide icon at `w-4 h-4`. Tooltip text: `"Clear feed"`.

### Clear Button Behavior

- **No confirmation dialog.** The feed is in-memory only; re-draining recovers the messages. An AlertDialog would add unnecessary friction for a dev tool.
- Action: calls `clearMessages()` Zustand action immediately on click.
- The feed header count updates to `"No messages"` immediately after clear.

### Accordion Feed (MessageFeedTab body)

Container: `ScrollArea className="flex-1 overflow-hidden"` (matches existing ResponseTab ScrollArea pattern).

Feed component: `Accordion type="single" collapsible className="w-full"` â€” one row open at a time, clicking the open row closes it.

**Collapsed row (AccordionTrigger):**
- Class: `px-4 py-2 text-xs font-mono` â€” compact single-line log-line style
- Content format: `{routingKey} â€˘ {exchange} â€˘ {contentType ?? "â€”"} â€˘ {formattedTimestamp}`
- Separator: bullet character `â€˘` (`&bull;`) with single space either side
- Timestamp format: `HH:MM:SS` via `new Date(timestamp * 1000).toLocaleTimeString([], { hour: '2-digit', minute: '2-digit', second: '2-digit', hour12: false })`. When `timestamp === null`: display `â€”` (em dash). Do NOT substitute `Date.now()`.
- Hover: default shadcn AccordionTrigger hover style (underline on hover is suppressed by the `text-xs font-mono` override â€” add `hover:no-underline` to AccordionTrigger className)

**Expanded row (AccordionContent):**
- Composes `<ResponseDecodedView decoded={msg.decoded} error={msg.error} />` followed by `<ResponseHexSection hexString={msg.hexString} decoded={msg.decoded} />`
- Both components receive props â€” they must NOT read from store (see RESEARCH.md Pitfall 1)
- Decode error case: `ResponseDecodedView` shows error inline (`"Decode failed: â€¦"` message) followed by `ResponseHexSection` showing raw hex â€” no error badge on the collapsed row (D-11)

**Row separator:** Each `AccordionItem` gets `border-b border-border`. No extra vertical spacing between items.

**Row hover highlight:** `hover:bg-muted/50` on AccordionTrigger â€” subtle, not high-contrast.

### Empty and Loading States

| State | What renders |
|-------|-------------|
| No messages yet (initial) | `<p className="text-xs text-muted-foreground p-4">Select a queue and click Drain</p>` |
| Drain in progress | Loader2 spinner inside disabled Drain button; feed content does not change during load |
| Drain returned 0 messages | `toast.info("Queue is empty")` â€” feed unchanged, no inline message |
| Drain partial error | `toast.error("Drain stopped early: {partial_error}")` â€” append partial results to feed |
| Feed cleared | Empty state renders immediately: `"Select a queue and click Drain"` |
| Decode error in row | Collapsed row shows metadata normally; expanded row shows `"Decode failed: â€¦"` + hex |
