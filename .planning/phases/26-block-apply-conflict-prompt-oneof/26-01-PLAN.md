       ```
       export type ConflictItem = {

       };
       ```

    4. Add ConflictChoices type: `export type ConflictChoices = Record<string, 'skip' | 'overwrite'>;`
       Compound key format: `{fieldName}:{collisionKey}` for map, `{fieldName}:{subFieldName}` for dirty-subfield, bare `{fieldName}` for branch-switch.

    5. Update ApplyBlockRef.commitApply signature: `commitApply: (plan: ApplyPlan, choices?: ConflictChoices) => void;`

    6. Add "oneof" to ELIGIBLE_KINDS set (per D-03).

    7. In buildApplyPlan, add a `conflicts: ConflictItem[] = []` accumulator alongside `toApply`.

    8. Replace the map-field block (lines 120-126) with per-key collision detection:
       - If field.kind.type === "map" and current is a non-empty array:
         - Build existingKeys Set from current rows: `new Set(current.map(r => String((r as {key: unknown}).key)))`
         - For each block row, check if its key collides:
           - Collision: push ConflictItem { fieldName: key, kind: 'map_key_collision', collisionKey: row.key, blockValue: row, currentValue: matching existing row }
           - No collision: accumulate into a local nonCollidingRows array
         - If nonCollidingRows.length > 0: push toApply item for key with value = nonCollidingRows array, kind "map"
         - If nonCollidingRows.length === 0 and all rows collide: do NOT push a toApply item
       - If field.kind.type === "map" and current is empty/undefined/null: existing behavior (push full value to toApply)

    9. Add oneof case AFTER the map case (per D-01, D-02, D-03, D-04, BLK-EXT-04, BLK-EXT-05):
       - Only execute when field.kind.type === "oneof"
       - Cast blockValue to object; read _selected: if absent or typeof !== 'string' → silent skip (continue)
       - Validate _selected against field.kind.branches: if not found → silent skip (continue), per D-02
       - Read currentSelectedBranch from formValues[key]?._selected (string | undefined)
       - If currentSelectedBranch and currentSelectedBranch !== blockBranch (branch-switch case per D-04/D-05):
         - Push ConflictItem { fieldName: key, kind: 'oneof_branch_switch', currentBranch: currentSelectedBranch, blockBranch: blockBranch, blockValue: blockValue, currentValue: formValues[key] }
         - continue (do not push to toApply)
       - Same-branch case (or no current branch yet):
         - Extract sub-field values from blockValue (all keys except _selected)
         - For each (subFieldName, subValue) in those entries:
           - Check dirty: cast dirtyFields[key] to object, check [subFieldName] === true
           - If dirty: push ConflictItem { fieldName: key, kind: 'oneof_dirty_subfield', subFieldName, blockValue: subValue, currentValue: (formValues[key] as any)?.[subFieldName] }
           - If clean: push toApply item { fieldName: `${key}.${subFieldName}`, value: subValue, kind: 'oneof' }

    10. Change the return: `return { toApply, conflicts, unknownKeys };`

    11. Update the JSDoc comment on buildApplyPlan to reflect Phase 26 behavior (add oneof + map-collision to the Rules section). Remove the "conflicts is always []" assertion.

    Pitfall D note: the `shouldDirty: false` concern is in ProtoFormRenderer.tsx (commitApply), NOT in this pure function. Do not add any setValue calls here.
