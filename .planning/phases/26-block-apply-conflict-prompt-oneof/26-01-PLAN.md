    2. Add ConflictItemKind type: export type ConflictItemKind = 'map_key_collision' | 'oneof_dirty_subfield' | 'oneof_branch_switch';

    3. Replace ConflictItem type with the extended version (all optional fields typed narrowly):
       - fieldName: string
       - blockValue: unknown
       - currentValue: unknown
       - kind: ConflictItemKind
       - subFieldName?: string          — oneof_dirty_subfield only
       - currentBranch?: string         — oneof_branch_switch only

       - nonCollidingBlockRows?: unknown[]  — map_key_collision only; carries the block rows that did NOT collide so commitApply Phase B can append them without a separate toApply item

       - subFieldLabel?: string         — display label for the sub-field in oneof_dirty_subfield rows

    4. Add ConflictChoices type: export type ConflictChoices = Record<string, 'skip' | 'overwrite'>;
       Compound key format: {fieldName}:{collisionKey} for map, {fieldName}:{subFieldName} for dirty-subfield, bare {fieldName} for branch-switch.

    5. Update ApplyBlockRef.commitApply signature: commitApply: (plan: ApplyPlan, choices?: ConflictChoices) => void;

    6. Add "oneof" to ELIGIBLE_KINDS set (per D-03).

    7. In buildApplyPlan, add a conflicts: ConflictItem[] = [] accumulator alongside toApply.

    8. Replace the map-field block with per-key collision detection. The CRITICAL invariant is: when any block row collides with an existing key, the entire map field is removed from toApply and its data flows exclusively through conflicts. This prevents the data-loss race where mapReplaceRegistry.replace() would destroy existing rows before Phase B can merge.

       Logic for field.kind.type === "map" and current is a non-empty array:
       - Build existingKeys Set: new Set((current as Array<{key: unknown}>).map(r => String(r.key)))
       - Partition block rows into collidingRows and nonCollidingRows arrays
       - If collidingRows.length > 0 (ANY collision):
         - Do NOT push a toApply item for this field
         - Push one ConflictItem per colliding block row:
           { fieldName: key, kind: 'map_key_collision', collisionKey: collidingRow.key, blockValue: collidingRow, currentValue: existingRowForKey, nonCollidingBlockRows: nonCollidingRows, fieldLabel: field.label ?? key }
         - Note: nonCollidingBlockRows is the SAME array reference on every ConflictItem for this field — commitApply groups by fieldName so it only needs to read it once (from conflicts[0] for that field)
       - If collidingRows.length === 0 (no collisions):
         - Push toApply item as before: { fieldName: key, value: blockRows, kind: 'map' }
       - If field.kind.type === "map" and current is empty/undefined/null: existing behavior (push full value to toApply, no conflicts)

    9. Add oneof case AFTER the map case (per D-01, D-02, D-03, D-04, BLK-EXT-04, BLK-EXT-05):
       - Only execute when field.kind.type === "oneof"
       - Cast blockValue to object; read _selected: if absent or typeof !== 'string' → silent skip (continue)
       - Validate _selected against field.kind.branches (find branch by name): if not found → silent skip (continue), per D-02
       - Read currentSelectedBranch from (formValues[key] as any)?._selected (string | undefined)
       - Branch-switch case: if currentSelectedBranch is a non-empty string AND currentSelectedBranch !== blockBranch (D-04/D-05):
         - Push ConflictItem { fieldName: key, kind: 'oneof_branch_switch', currentBranch: currentSelectedBranch, blockBranch: blockBranch, blockValue: blockValue, currentValue: formValues[key], fieldLabel: field.label ?? key }
         - continue (do not push to toApply)
       - Same-branch case (currentSelectedBranch === blockBranch, or no current branch yet):
         - Extract sub-field entries from blockValue: all keys except '_selected'
         - For each (subFieldName, subValue) in those entries:
           - Check dirty: (dirtyFields[key] as Record<string, unknown> | undefined)?.[subFieldName] === true
           - If dirty: push ConflictItem { fieldName: key, kind: 'oneof_dirty_subfield', subFieldName, blockValue: subValue, currentValue: (formValues[key] as any)?.[subFieldName], fieldLabel: field.label ?? key, subFieldLabel: subFieldName }
           - If clean: push toApply item { fieldName: `${key}.${subFieldName}`, value: subValue, kind: 'oneof' }

    10. Change the return: return { toApply, conflicts, unknownKeys };

    11. Update the JSDoc comment on buildApplyPlan to reflect Phase 26 behavior (add oneof + map-collision to the Rules section). Remove the "conflicts is always []" assertion. Add a note: "Map fields with any collision go entirely through conflicts (nonCollidingBlockRows carried on ConflictItem) — no toApply item is emitted."

    Pitfall D note: shouldDirty: false is a concern of ProtoFormRenderer.tsx (commitApply), NOT this pure function. Do not add any setValue calls here.
