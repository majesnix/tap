## Spacing Scale

Declared values (multiples of 4). All in use by existing field components.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon margin inside button (`mr-1`) |
| sm | 8px | Row internal padding (`p-2`), row gap (`gap-2`) |
| md | 16px | Section padding, form stack gap |
| lg | 24px | Section breaks |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- `mb-3` (12px) used in RepeatedField for bottom margin of the entire field block — retain for visual continuity with existing form fields. Not on the 8pt scale; accepted as inherited pattern.
- Row container uses `flex items-start gap-2 p-2 border rounded` — `items-start` is intentional to support variable-height value renderers (nested message sub-forms grow downward without misaligning the Remove button).

## Layout Contract

### Map Field Header Row

```
[Badge: "map" + key/value type annotation]  [Label: field.label]
```

- Badge: `<Badge variant="secondary" className="text-xs">map&lt;K, V&gt;</Badge>` where K and V are the declared proto types (e.g. "map&lt;string, int32&gt;").
- Label: `<span className="text-sm font-semibold">{field.label}</span>`.
- Container: `<div className="flex items-center gap-2">` — mirrors RepeatedField header.

### Key-Value Row

```
[Key input (flex-1 max)] [Value renderer (flex-1)] [Remove button (shrink-0)]
```

- Row container: `<div className="flex items-start gap-2 p-2 border rounded">`.
- Key column: fixed-width constraint `className="w-1/3"` — prevents value renderer from crowding the key.
- Value column: `className="flex-1"`.
- Remove button: `className="shrink-0 mt-1"` — top-aligned (same as RepeatedField).

### Duplicate Key Error

Rendered immediately below the affected key input, inside the key column:

```tsx
<p className="text-xs text-destructive mt-1" role="alert">Duplicate key</p>
```

Shown for every row that shares a duplicate key (not just the later-added row). Matches ScalarField error display pattern.

### Hidden Duplicate Guard

- Field name: `${path}.__mapDuplicateGuard`
- Not rendered visually. Keeps `formState.isValid` false while duplicates exist.
- Implemented as a hidden `Controller` or `register` with a validate rule.

### Add Entry Button

```
[Plus icon] Add entry
```

- Placed below the row list: `className="self-start"`.
- Button: `<Button type="button" variant="outline" size="sm">`.
- Icon: `<Plus className="w-4 h-4 mr-1" />`.

### Empty State (Zero Rows)

No separate empty-state placeholder. When the field has zero rows, only the field header and the "Add entry" button are visible. This matches RepeatedField precedent — the Add button is the affordance.

## Interaction Contract

### Key Input Behavior by Key Type

| Proto Key Type | Input Mechanism | Constraint |
|----------------|-----------------|------------|
| `string` | `<Input type="text" />` | None |
| `int32`, `uint32`, `sint32`, `fixed32`, `sfixed32` | `<Input type="number" />` | Step 1, no decimals |
| `int64`, `sint64`, `sfixed64` | `<Input type="text" />` | Regex `/^-?[0-9]+$/` — negative sign allowed |
| `uint64`, `fixed64` | `<Input type="text" />` | Regex `/^[0-9]+$/` — no negative |
| `bool` | `<Select>` with "true" / "false" | Exactly 2 options; at most 2 map entries possible |

Source: CONTEXT.md `<decisions>` D-09, D-10, D-11.

### Validation Trigger

- Form mode: `mode: onBlur` (project-wide, from STATE.md key decisions).
- Duplicate detection: fires on `onChange` of any key input — not deferred to blur or submit. Inline `<p role="alert">` updates immediately.
- Send block: active while `hasDuplicates === true` via hidden guard field. `formState.isValid` is false.

### Value Renderer

- Rendered via `renderValue: RenderFieldFn` prop — the same pattern as `RepeatedField.renderItem`.
- Accepts `BytesField` (Phase 6), `ScalarField`, `EnumField`, `NestedMessageField` — all existing renderers.
- `renderValue` is `ProtoFormRenderer.renderField` passed in from the parent. No direct import of `ProtoFormRenderer` inside `MapField` (avoids circular import).

### Row Management

- New rows appended at bottom via `append()`.
- Removed rows deleted immediately via `remove(index)` — no confirmation.
- Rows keyed by `rhfField.id`, never by array index (RHF `useFieldArray` requirement).
- Default key value when adding a new row: `""` (string), `0` (numeric), `"false"` (bool).

## Accessibility Contract

- Remove entry button: `aria-label="Remove entry"` (icon-only button, label required).
- Add entry button: text label "Add entry" visible (no `aria-label` needed).
- Duplicate key error: `role="alert"` so screen readers announce on change.
- Bool key Select: uses shadcn `Select` which follows Radix `SelectRoot` accessible combobox pattern.
- Key input for 64-bit text fields: includes `aria-describedby` pointing to the error paragraph when the error is shown.
