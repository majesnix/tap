  <how-to-verify>
**Prerequisites:** RabbitMQ running locally (or accessible), Tauri app dev server running via `npm run tauri dev` in the project root.

**Step 1 — Prepare a test .proto file** with a map field:

```protobuf
syntax = "proto3";

message TestMap {
  map<string, int32> labels = 1;
  map<bool, string> flags = 2;
}
```

Save as `test_map.proto` in any directory.

**Step 2 — Load the proto file:**
- Open the app, load `test_map.proto` using the file picker
- Select the `TestMap` message type
- Confirm the form renders two map fields: "Labels" (map badge showing `map<string, int32>`) and "Flags" (map badge showing `map<bool, string>`)

**Step 3 — Verify Add/Remove (MFLD-01):**
- Click "Add entry" on the Labels field → a row appears with a key text input and a value number input
- Click "Add entry" again → a second row appears
- Click the Remove (trash) button on the second row → the row disappears

**Step 4 — Verify key type constraints (MFLD-02):**
- Labels field key input: type text → ✓ (string key type)
- Labels field value input: type a number → ✓ (int32 value)
- Flags field: click "Add entry" → the key column shows a Select dropdown with "true" and "false" options (not a text input)

**Step 5 — Verify duplicate key error (MFLD-03):**
- On the Labels field, add two rows and type "x" in both key inputs
- Expected: "Duplicate key" error appears below each affected key input immediately (on change, not on blur)
- Expected: The Send button is disabled while duplicates exist
- Fix the duplicate (change one key to "y") → the error disappears → the Send button becomes enabled

**Step 6 — Verify value renderers (MFLD-04):**
- For Labels field: the value column renders a number input (scalar int32 renderer)
- For Flags field: add an entry, value column renders a text input (scalar string renderer)
- If your test proto has a message-valued map (e.g. `map<string, MyNestedMessage>`), verify the value column renders the nested message sub-form

**Step 7 — Verify binary send (MFLD-05):**
- Connect to RabbitMQ, select a queue or exchange
- Fill in Labels with at least one entry (e.g. key="env", value=42)
- Click Send
- Expected: message is accepted (no error dialog), appears in RabbitMQ queue
- In RabbitMQ management UI (http://localhost:15672), find the message in the queue and verify it is binary (protobuf wire format — not plain text JSON)
  </how-to-verify>
