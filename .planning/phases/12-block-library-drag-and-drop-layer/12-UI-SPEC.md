## Interaction Contract: Drag-and-Drop

This section defines the visual and interaction states specific to Phase 12.

### Drag Source (Block List Rows)

| State | Class | Behavior |
|-------|-------|----------|
| Default | `cursor-grab` | Visual hint that rows are draggable |
| Active drag in progress | `active:cursor-grabbing` | Native DnD provides ghost image; source row appearance unchanged (no opacity reduction) |
| Editor view | not rendered | Editor view shows no block list rows — no drag source exists naturally |

Implementation:
- `draggable="true"` on each block list row `<div>` in list view only
- `onDragStart`: `e.dataTransfer.setData('blockId', block.id)` — block ID only, not content
- Add `cursor-grab active:cursor-grabbing` to existing row class string alongside `hover:bg-muted`

### Drop Zone (FormPanel ScrollArea)

| State | Visual Change | Trigger |
|-------|--------------|---------|
| Default | No visual change | — |
| Drag entering / over | `ring-2 ring-primary/50` applied to the ScrollArea div | `onDragOver` sets `isDraggingOver = true` |
| Drag leaving | Ring removed | `onDragLeave` sets `isDraggingOver = false` |
| Drop completed | Ring removed + merge executes | `onDrop` clears `isDraggingOver`, calls `applyBlockRef.current()` |
| JSON editor mode | No drop zone rendered | `<ScrollArea>` wrapping `ProtoFormRenderer` is not mounted in JSON mode — naturally no drop target |

`isDraggingOver` is local `useState<boolean>` in `FormPanel` — not in Zustand store.

Drop zone boundary: the `<ScrollArea>` wrapping `ProtoFormRenderer` only (line 237 of FormPanel.tsx). The FormPanel header, JSON editor, and publish bar are not drop targets.

### Block Merge Behavior

| Case | Outcome | Toast |
|------|---------|-------|
| Block key matches top-level scalar/enum field AND field is not dirty | `setValue(key, value)` fills the field | None |
| Block key matches top-level scalar/enum field AND field is dirty (user-edited) | Field is NOT overwritten | None — intentional behavior, not a warning |
| Block key has no matching top-level scalar/enum field (nested, map, repeated, or absent) | Key is skipped | Warning toast (see Copywriting) |
| All block keys skipped (block has zero fillable fields matching the schema) | All keys listed in warning | Warning toast listing all keys |
| Block JSON parse fails at apply time | Silent no-op | None — unreachable path since Save validated JSON; fail silently |
| Drop succeeds with all fields already dirty or no schema loaded | Silent — nothing filled, no toast (no matching fields were attempted) | None |

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Primary CTA | N/A — drag-and-drop interaction; no button | This phase |
| Empty state heading | "No blocks yet" | Inherited from Phase 11 (BlockLibraryPanel.tsx line 184) |
| Empty state body | "Save JSON snippets you can reuse across messages." | Inherited from Phase 11 (BlockLibraryPanel.tsx line 187) |
| Warning toast — skipped fields | `"{N} field(s) from block not in form: {fieldA}, {fieldB}, …"` | CONTEXT.md D-06 |
| Warning toast — pluralization | "field" when N=1; "fields" when N>1 | Match FormPanel.tsx JSON-mode pattern (line 158) |
| Destructive confirmation | N/A — no destructive actions in this phase | This phase |
| Error state | N/A — parse failure is silent no-op; merge errors surface only as skipped-field warnings | This phase |

Warning toast implementation: `toast.warning(\`${n} field(s) from block not in form: ${skipped.join(', ')}\`)` — fires only when `skipped.length > 0`. Uses `sonner` already imported in `BlockLibraryPanel.tsx`.

Toast truncation: accept Sonner's default display behavior. No explicit max-N truncation rule.

## Accessibility Notes

- Drop zone does not need an ARIA drop role for a developer tool — native DnD is sufficient
- `cursor-grab` / `cursor-grabbing` is a visual-only affordance; no ARIA attribute change needed for draggable rows
