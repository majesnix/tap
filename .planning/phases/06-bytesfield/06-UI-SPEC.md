## Interaction Contract

### Validation Trigger

- Fires on **blur** — consistent with `mode: onBlur` set project-wide on `useForm`
- Does NOT fire on change (no live validation while typing)
- Validation covers:
  1. Character set check: regex `/^[A-Za-z0-9+/]*={0,2}$/` (catches URL-safe `-` / `_` per BFLD-03)
  2. Structural check: `.refine()` with `atob()` (catches structurally invalid lengths per BFLD-04)
  - Both use the same error message (D-08)

### Popover State

- Controlled via local `useState` (`popoverOpen` / `setPopoverOpen`)
- Opens when user clicks "From text"
- Closes automatically when user clicks "Convert" (same action — D-04)
- Close-without-convert: user can close via outside click (Popover default behavior)
- `utf8Input` local state is cleared on Convert action

### Byte Count Display

- Condition: `!fieldState.error && rhfField.value !== ""`
- Content: `{atob(rhfField.value).length} bytes` (always-plural format — consistent with success criterion example "14 bytes")
- `1 bytes` is the intended output for a single-byte value (no plural logic required)
- Position: immediately below `<Input>`, before the error message position, before the popover trigger

### Error Display

- Position: below Input (and below byte count when present — byte count and error are mutually exclusive states)
- Format: `{field.label}: {fieldState.error.message}`
- Element: `<p className="text-xs text-destructive" role="alert">`

### Layout Order (top to bottom, single column)

```
[Label row: <Label> + <Badge variant="outline">bytes</Badge>]
[<Input type="text" placeholder="base64 encoded value">]
[byte count: <p class="text-xs text-muted-foreground">{n} bytes</p>]  ← visible only when valid + non-empty
  OR
[error: <p class="text-xs text-destructive" role="alert">...</p>]    ← visible only when error
[<Button variant="outline" size="sm">From text</Button>]              ← always visible, left-aligned
```

## Copywriting Contract

> Note: This phase adds a single form field component, not a new screen. The "Primary CTA" is the popover Convert action. There is no page-level primary action, empty state (list/table context), or destructive confirmation in this phase.

| Element | Exact Copy | Notes |
|---------|-----------|-------|
| Primary CTA (popover) | `Convert` | Popover action button; closes popover + fills field |
| Field type badge | `bytes` | Replaces old `bytes (base64)` badge from ScalarField (D-02) |
| Input placeholder | `base64 encoded value` | Shown when input is empty |
| Helper button label | `From text` | Exact match for BFLD-02 wording |
| Popover textarea placeholder | `Type UTF-8 text to convert to base64…` | Includes ellipsis |
| Validation error (BFLD-03 + BFLD-04) | `Must be valid base64 (standard alphabet, not URL-safe)` | Single message for both error classes (D-08) |
| Error display format | `{field.label}: {errorMessage}` | Matches ScalarField pattern, e.g. `image_data: Must be valid base64 (standard alphabet, not URL-safe)` |
| Byte count label | `{n} bytes` | Always-plural; shown only when valid + non-empty; e.g. `14 bytes` |

**Empty state:** Not applicable — this phase introduces a single form input field. The closest analogs are the Input and Textarea placeholders (listed above).

**Destructive confirmation:** None in this phase — no destructive actions introduced.
