## New Component: ThemeToggle

This is the only net-new UI component in Phase 5.

### Anatomy

```
<Button variant="ghost" size="icon" className="size-8">
  {icon at size-4: Monitor | Sun | Moon}
</Button>
```

### State Machine

| State | Icon | aria-label | title |
|-------|------|-----------|-------|
| system (current mode) | Monitor | "System theme" | "System theme" |
| light (current mode) | Sun | "Light theme" | "Light theme" |
| dark (current mode) | Moon | "Dark theme" | "Dark theme" |
| pre-mount | disabled Button, size-8 | (none — disabled) | (none) |

Cycle order on each click: system → light → dark → system (wrap). The `aria-label` and `title` describe the **current** mode (not the next mode) — this tells the user what is active, matching the icon shown.

### Pre-mount Contract

Before React hydration completes, `theme` is `undefined`. Return a `<Button variant="ghost" size="icon" className="size-8" disabled />` placeholder to prevent layout shift. The placeholder must occupy identical dimensions to the live button.

### Placement

Sidebar footer — replaces the existing centered `<div className="text-xs text-muted-foreground text-center">` with:

```tsx
<div className="flex items-center justify-between">
  <div className="text-xs text-muted-foreground">v0.1.0 — Walking Skeleton</div>
  <ThemeToggle />
</div>
```

Source reference: `src/components/sidebar/Sidebar.tsx` lines 57–61.

### Visual Requirements

- Button must not have visible background at rest (ghost variant)
- Button hover: `--accent` fill (automatic via ghost variant)
- Button size: 32px × 32px (`size-8`)
- Icon size: 16px × 16px (`size-4`)
- No border, no shadow at rest
- Focus ring uses `--ring` token (automatic via shadcn button)
- Unobtrusive — must not compete visually with the version string

## ThemeProvider Architecture Contract

This section exists so the executor implements the correct pattern without ambiguity.

### Wrapping Order in App.tsx

```
ThemeProvider (attribute="class", defaultTheme="system", enableSystem)
  ThemeBootstrap     ← must be INSIDE ThemeProvider
  AppLayout
  Toaster
```

`ThemeBootstrap` must be a child of `ThemeProvider`. Placing it as a sibling makes `useTheme()` return a stub and `setTheme()` becomes a no-op.

### Persistence Bridge Contract

- On startup: read `tauri-plugin-store` key `"theme-mode"` → call `setTheme(saved)` to override localStorage → set `bootstrapped = true`
- On theme change: only mirror to `tauri-plugin-store` when `bootstrapped === true` and `theme !== undefined`
- The `bootstrapped` guard is non-negotiable (prevents the startup race described in RESEARCH.md Pitfall 6)

Store file: `"proto-sender.json"` (same file used by connection profiles — established project pattern).
Store key: `"theme-mode"`.
