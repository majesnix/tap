## Interaction Contracts

### Clear Button (FRM-01 + KB-03)

**Placement:** FormPanel header right side, leftmost of the existing icon-button group (before Library toggle, before JSON toggle)

**Visual spec:**
- `variant="ghost" size="icon-sm"` — matches Library and Braces buttons exactly
- Icon: `RotateCcw` (lucide), `size={16}` — matches `Library` icon size in same row
- No destructive color; ghost variant at rest, `hover:bg-muted` on hover
- Tooltip: platform-branched — `"Clear form (⌘⇧R)"` on macOS, `"Clear form (Ctrl+Shift+R)"` on Windows/Linux
- `aria-label`: `"Clear form"`
- **TooltipProvider pattern:** wrap the button with a local `<TooltipProvider>` as the established pattern in this codebase (e.g. `PublishBar.tsx`, `PlanRunBar.tsx`). No global provider exists at the app root.

**Behavior:**
- Single click: no confirmation prompt (D-08 is locked)
- Exits JSON mode if active (`setIsJsonMode(false)`), then triggers `setPendingReplayValues(buildDefaultValues(message))`
- JSON draft string is discarded silently
- The `setPendingReplayValues` path is MANDATORY — never call `resetRef.current()` directly

### Hover-Reveal Copy Icon (FRM-02)

**Applies to:** ScalarField, EnumField, BytesField — NOT repeated, map, nested message, or oneof wrapper fields (D-13)

**Field row layout:**

```
[Label text]  [type badge]          [Copy icon — hidden at rest]
[Input / Select / Textarea control]
```

The label row uses `flex items-center justify-between` (or add `ml-auto` on the copy icon container). The existing label rows use `flex items-center gap-2` for `[Label] [Badge]` — the copy icon sits at the trailing edge using `ml-auto` on a wrapper `div`, not a new `justify-between` that would break the existing label/badge spacing.

The copy icon appears in the label row trailing edge, visible only on field row hover.

**Visual spec:**
- Outer field `div` (the `flex flex-col gap-1 mb-3` wrapper) gets an additional `group` Tailwind class
- Copy icon button wrapper: `ml-auto` on a `div` containing the Tooltip + Button
- Copy icon button: `variant="ghost" size="icon-sm"` with `opacity-0 group-hover:opacity-100 transition-opacity` classes
- Icon: `Copy` (lucide), `size-3` (12px) — matches `text-xs` badge scale
- On success state: swap `Copy` icon to `Check` icon for exactly 1500ms, then revert to `Copy` icon
- `Check` icon color: `text-foreground` (not accent, not green)
- Tooltip on the icon button: `"Copy value"` — use local `<TooltipProvider>` wrapping the copy button (established pattern)

**Copy format per field type (D-11):**
- `string` → value as-is
- `int64` / `uint64` → string representation (e.g. `"123456789012345"`)
- `bool` → `"true"` or `"false"`
- `enum` → string name of the selected enum value (e.g. `"STATUS_OK"`)
- `bytes` → base64 string as currently stored in form state

**Clipboard error handling:**
- If `navigator.clipboard.writeText()` rejects (browser policy, iframe context): show Sonner toast `"Copy failed — clipboard not available"` using the existing `toast.warning()` pattern from FormPanel. No icon-swap to Check; icon remains in Copy state.

### Keyboard Shortcuts (KB-01–KB-04)

All shortcuts use `react-hotkeys-hook@^5.3.2` (D-01). Platform symbol detection uses `navigator.userAgent` regex — the established pattern from `Sidebar.tsx` (`/Macintosh|MacIntel|MacPPC|Mac68K/.test(navigator.userAgent)`).

| Shortcut (macOS) | Shortcut (Win/Linux) | Action | Inhibit rule | Registration point |
|-----------------|---------------------|--------|-------------|-------------------|
| Cmd+Enter | Ctrl+Enter | Send form | None — fires everywhere including form inputs and CodeMirror | FormPanel `useHotkeys` + CodeMirror keymap extension (D-06) |
| Cmd+O | Ctrl+O | Open proto file picker | None — fires globally | FormPanel or AppLayout `useHotkeys` |
| Cmd+Shift+R | Ctrl+Shift+R | Clear form | Inhibited in `<input>`, `<textarea>`, contenteditable (`enableOnFormTags: false`) | FormPanel `useHotkeys` |
| Cmd+1 | Ctrl+1 | Switch to Hex tab | Inhibited in form elements (`enableOnFormTags: false`) | RightPanel or AppLayout `useHotkeys` |
| Cmd+2 | Ctrl+2 | Switch to History tab | Inhibited in form elements (`enableOnFormTags: false`) | RightPanel or AppLayout `useHotkeys` |
| Cmd+3 | Ctrl+3 | Switch to Response tab | Inhibited in form elements (`enableOnFormTags: false`) | RightPanel or AppLayout `useHotkeys` |

**Cmd+Enter CodeMirror dual registration (D-06):**
- Window-level `useHotkeys` handler must check `event.target.closest('.cm-editor')` — if true, skip (CodeMirror keymap handles it)
- CodeMirror keymap extension injected into `JsonEditor` as a prop (e.g. `keymapExtension` prop) — fires the same send action

**Tab targets for Cmd+1/2/3:** RightPanel tabs `"hex"` (1), `"history"` (2), `"response"` (3) — sourced from `RightPanel.tsx` `activeTab` state values.

### Shortcut Tooltip Format (KB-05)

Every button that has a keyboard shortcut MUST show the shortcut in its tooltip content. Use local `<TooltipProvider>` wrapping pattern (not a global provider) — established pattern in `PublishBar.tsx`, `PlanRunBar.tsx`, `StepStatusBadge.tsx`, `MessageFeedTab.tsx`, `ResponseQueuePicker.tsx`.

**Tooltip composition for TabsTrigger:** wrap `<TabsTrigger>` with `<TooltipTrigger asChild>` to pass the ref correctly through Radix composition.

| Button | macOS tooltip | Windows/Linux tooltip |
|--------|--------------|----------------------|
| Send (PublishBar) | `"Send (⌘↵)"` | `"Send (Ctrl+Enter)"` |
| Open proto file | `"Open proto file (⌘O)"` | `"Open proto file (Ctrl+O)"` |
| Clear form | `"Clear form (⌘⇧R)"` | `"Clear form (Ctrl+Shift+R)"` |

Tab shortcut tooltips (on the RightPanel TabsTrigger elements):

| Tab | macOS tooltip | Windows/Linux tooltip |
|-----|--------------|----------------------|
| Hex | `"Hex (⌘1)"` | `"Hex (Ctrl+1)"` |
| History | `"History (⌘2)"` | `"History (Ctrl+2)"` |
| Response | `"Response (⌘3)"` | `"Response (Ctrl+3)"` |
