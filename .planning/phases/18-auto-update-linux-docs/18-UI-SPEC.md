## Update Notification Toast — Interaction Contract

This is the only net-new UI element in Phase 18.

### Trigger
- Fires once on app startup (React `useEffect` with `[]` deps in `App.tsx`)
- Appears only when `check()` returns an update with `available: true`
- Silent if no update available (no UI change, no toast)
- Silent on error (console.error only; no toast, no modal)

### Position
- Bottom-right (Sonner default — `Toaster` in `src/components/ui/sonner.tsx` does not set a `position` prop)

### Toast anatomy

| Element | Value |
|---------|-------|
| Title | `Update {update.version} available` |
| Description | `update.body` if non-null; otherwise `"A new version is ready to install."` |
| Action label | `Install & Relaunch` |
| Duration | `Infinity` — toast persists until user acts or manually dismisses |
| Type | Plain (not `toast.success`, `toast.error`, etc.) — neutral informational style |
| Dismissable | Yes — Sonner renders a close button by default |

### Action flow on click

1. User clicks "Install & Relaunch"
2. `update.downloadAndInstall()` is called (handled by `tauri-plugin-updater`)
3. On success: `relaunch()` is called (handled by `tauri-plugin-process`)
4. On error: `console.error('Install failed:', err)` — no secondary toast, no modal

### React 18 strict-mode note
In development, the `useEffect` fires twice. The `check()` call is idempotent; this
produces at most one toast per app session in production (StrictMode fires twice in dev
but once in production).
