<objective>
Generate the Ed25519 signing keypair, wire the public key into tauri.conf.json, guide the developer through uploading the private key to GitHub secrets, then verify the end-to-end signing pipeline by triggering a workflow and confirming latest.json is complete.

Purpose: This is the final activation step for auto-update. Plans 01-03 wired the infrastructure; Plan 04 provides the cryptographic material that makes signing work and verifies the complete pipeline end-to-end.

Pre-condition: Phase 17 must be complete (Apple signing secrets — APPLE_CERTIFICATE, APPLE_CERTIFICATE_PASSWORD, APPLE_ID, APPLE_ID_PASSWORD, APPLE_TEAM_ID — must already be set in the GitHub repo). The Task 3 tag-push verify path triggers the macOS job unconditionally; without Apple secrets the macOS build fails and the "both jobs green" criterion cannot be met.

Output: Real pubkey in tauri.conf.json; GitHub secrets populated; signed release artifacts with latest.json accessible at the update endpoint.
</objective>

  <files>src-tauri/tauri.conf.json</files>

  </read_first>
  <action>
**Step 1 — Check if keypair already exists:**

```bash
test -f ~/.tauri/tap.key && echo "Key already exists — skip generation" || echo "Key not found — generating"
```

If the key already exists at `~/.tauri/tap.key`, skip generation and read the public key directly from `~/.tauri/tap.key.pub`. Do NOT regenerate if a key exists — this would invalidate any already-deployed releases.

**Step 2 — Generate keypair (only if not already existing):**

```bash
pnpm tauri signer generate -w ~/.tauri/tap.key
```

When prompted for a password/passphrase, press Enter twice for no passphrase (simpler secret management). Store the passphrase securely if one is entered.

This creates:
- `~/.tauri/tap.key` — Ed25519 private key (NEVER commit this file)
- `~/.tauri/tap.key.pub` — Ed25519 public key

**Step 3 — Read the public key content:**

```bash
cat ~/.tauri/tap.key.pub
```

The output will look like:
```
untrusted comment: minisign public key XXXXXXXXXXXXXXXX
RWTXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
```

Copy the FULL multi-line content (both lines including the `untrusted comment:` line).

**Step 4 — Replace the placeholder in tauri.conf.json:**

In src-tauri/tauri.conf.json, replace the `"pubkey"` value `"UPDATER_PUBKEY_PLACEHOLDER"` with the exact content of `~/.tauri/tap.key.pub` as a single-line JSON string (escape newlines as `\n`).

The final pubkey value in tauri.conf.json must be:
```json
"pubkey": "untrusted comment: minisign public key XXXXXXXXXXXXXXXX\nRWTXXXX..."
```

The pubkey is a public key — it is safe to commit to the repository. Only the `.key` file (private key) must stay out of version control.

CRITICAL SAFETY CHECK: Verify you are writing `~/.tauri/tap.key.pub` content (public) into tauri.conf.json, NOT `~/.tauri/tap.key` (private). The public key file ends in `.pub`. The private key file has no extension.
  </action>

  </verify>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ~/.tauri/tap.key → GitHub secret | Private key traverses local filesystem to browser/GitHub UI — this is a high-value secret; treat as equivalent to a production deploy key |
| tauri.conf.json pubkey → repo | Public key is public; committing it to the repo is intentional and correct |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-12 | Information Disclosure | ~/.tauri/tap.key private key | mitigate | Private key stored ONLY in GitHub Actions secret; never staged in git; .gitignore should cover ~/.tauri/ (outside repo root anyway); offline backup responsibility is developer's |
| T-18-13 | Tampering | latest.json platform entry injection | mitigate | Signature verification in tauri-plugin-updater covers the binary artifact, not the JSON; an injected JSON entry cannot execute unsigned code |
| T-18-14 | Information Disclosure | private key in tauri.conf.json (Pitfall 6) | mitigate | acceptance_criteria explicitly checks grep for "UPDATER_PUBKEY_PLACEHOLDER" and for "untrusted comment:" — the latter is the signature of a public key, not a private key |
| T-18-15 | Denial of Service | draft release blocks update endpoint | accept | Documented and accepted: releaseDraft:true requires manual publish step; update endpoint returns 404 until publish; check() handles 404 silently |
</threat_model>
