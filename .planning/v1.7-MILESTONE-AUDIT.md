## 5. E2E Flow Traces

### Flow 1: Drag block, no conflicts ✅ COMPLETE

```
useDndMonitor.onDragEnd → event.over.id === 'form-drop-zone'
→ blockValues parsed from block JSON
→ applyBlockRef.current.buildPlan(blockValues) → ApplyPlan { toApply, conflicts: [] }
→ plan.conflicts.length === 0 → commitApply(plan) immediately
→ Phase A: setValue(fieldName, value, { shouldDirty: false }) / mapReplaceRegistry for maps
→ RHF form state updated
```

### Flow 2: Drag block, map key collision ✅ COMPLETE

```
buildPlan → plan.conflicts: [map_key_collision, ...]
→ setConflictPlan(plan); setConflictChoices({})
→ AlertDialog opens (conflictPlan !== null)
→ user reviews rows (RadioGroup Skip/Overwrite per conflict)
→ Apply → commitApply(conflictPlan, conflictChoices)
→ Phase B map: mergedExisting (collide: overwrite or preserve) + nonCollidingBlockRows
→ mapReplaceRegistry.current[fieldName](merged) → single replace() call
→ setConflictPlan(null) → dialog closes
```

### Flow 3: Drag block, oneof branch switch ✅ COMPLETE

```
buildPlan → plan.conflicts: [oneof_branch_switch]
→ dialog opens → user sets Overwrite
→ Apply → Phase B: single atomic setValue({ _selected: blockBranch, [blockBranch]: subValue }, { shouldDirty: false })
→ OneofField re-renders with new branch, all fields clean
```

### Flow 4: History search with AND filters ✅ COMPLETE

```
Search input → onSearchChange → searchQuery state → 200ms debounce → debouncedSearch
→ useMemo: filterHistoryEntries(entries, typeFilter, targetFilter, debouncedSearch)
→ Three chained .filter() calls (AND logic) → filteredEntries
→ HistoryTable renders filtered results; count label: isFiltered ? "X of Y / 100" : "Y / 100"
```

### Flow 5: History field-name search ✅ COMPLETE

```
User types field name (e.g. "orderId")
→ filterHistoryEntries → collectSearchTokens(e.fieldValues)
→ tokens include "orderId" key → token.toLowerCase().includes(q) → match
→ entry remains in filteredEntries
```

## 6. Tech Debt

### Phase 24

- **`collectFieldNames` dead export** (INFO): `historyHelpers.ts` exports `collectFieldNames` (returns field name keys only). Live `filterHistoryEntries` uses `collectSearchTokens` (returns keys + primitive values). The correct helper is wired; `collectFieldNames` is only used in tests. Suggestion: remove the export or document the distinction.

### Phase 25

- **`replace()` marks dirty** (INFO — accepted): `useFieldArray.replace()` in `MapField.tsx` marks the field dirty (RHF 7.76.1 limitation). Block-applied maps are intentionally "user-owned" after first fill. Phase 26 conflict dialog handles the second drag via `plan.conflicts`. No fix required.
- **SUMMARY test count discrepancy** (INFO): Phase 25 SUMMARY.md reported 997/16 test counts vs actual 499/8. Documentation artifact — no code defect.

### Phase 26

- **15 human runtime scenarios** (INFO): All 15 verification scenarios (Phase 24: 4, Phase 25: 6, Phase 26: 5) require the running Tauri app. These were UAT-verified by the developer (UAT results documented in `*-HUMAN-UAT.md` files). Code analysis cannot substitute for runtime verification of dialog lifecycle, RHF field-array state, and WKWebView DnD behavior.
