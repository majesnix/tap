<objective>
Fix message-kind fields (e.g. "items", "shipping", "payment") appearing in the "not in form" toast when a block is dropped onto the form.

Purpose: The toast is meant to surface genuinely unknown keys — block keys that have no matching field in the schema. Message-kind fields do exist in the schema; they are intentionally ineligible. These two buckets must be separated so the toast fires only for true unknowns.

Output: `ApplyPlan` gains `unknownKeys: string[]`; `buildApplyPlan` populates it correctly; `FormPanel` uses it for the toast; tests cover the separation.
</objective>
