In `src-tauri/tauri.conf.json`, under `app.security`, replace `"csp": null` with the following EXACT policy string (single-line JSON string literal, do not alter, reorder, or reformat the directives):

`default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self'; connect-src 'self' ipc: http://ipc.localhost; object-src 'none'; base-uri 'self'; frame-ancestors 'none'`

Do not escape quotes — the policy uses only single quotes internally so it embeds cleanly in a double-quoted JSON value.

Rationale (informational — do not change the policy): Tauri 2 auto-appends script nonces/hashes for bundled code at compile time so `script-src 'self'` works; `style-src 'unsafe-inline'` is intentionally required by Radix UI / shadcn inline positioning styles; `connect-src ipc: http://ipc.localhost` is the documented Tauri 2 IPC requirement for `invoke`. The frontend makes no network calls (Management API + AMQP are Rust-side).

Contingency (apply ONLY if `tauri dev` HMR breaks under this CSP — do not apply by default and do not weaken the production policy): the documented mitigation is appending `ws://localhost:1420 http://localhost:1420` to the `connect-src` directive for the dev configuration only. Leave the production policy above unchanged.
